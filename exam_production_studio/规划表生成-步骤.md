# 规划表生成流程建立 · 步骤文档

> 目标：让 studio 的规划表链路对齐两份权威《规划表编写说明.md》，支持
> **一课一练**（8 列单表，目录驱动，三行标题）与 **考纲百套卷**（10 列总表 + 映射表 + 细目表）两种驱动，
> 并采用「LLM 生成内容 + 代码渲染/校验 + 人工确认」的自动生成方案。
> 背景与差距诊断、目标 schema、自动生成方案见《Q：规划表生成.md》§九 / §十。
>
> 原则：先打通「结构化数据 → 数据契约 → 生成/校验」，命名与逐卷配置随后；改动尽量为「加法」，不破坏现有流程。
>
> 相关既有代码：planning `backend/engine/steps/planning.py`；mapping `backend/engine/steps/mapping.py`；
> 映射解析 `backend/shared/xueke_api/kpoint_resolver.py`；LLM 单一入口 `backend/shared/ai/llm.py::call_api`；
> AI 留档 `backend/shared/ai/trace.py`；OCR `backend/shared/ocr/scanner.py`；命名 `backend/shared/docx/naming.py`；
> 落库 `backend/db.py`；编排 `backend/engine/runner.py`。

## 决策口径（已与用户确认，2026-07-02）

- 步骤文档定位：**studio 实现步骤计划**（非内容生产 SOP）。
- **不强制上传标准 xlsx**：既可解析上传，也可（文字版/图片版）PDF 目录驱动合成。
- **LLM 承担内容 + 卷号编排 + 映射**；准确性由「代码校验器（硬拦截）+ 人工确认闸门」保证，非靠限制 LLM 范围。
- 自动化程度：**生成 + 人工确认**，不达标行标红回退重生成。
- 样本与权威说明严格一致，样本作为目标 schema 与 few-shot 提示来源（需先转文本表）。
- **参照 Agent 全能体做法，落成两段式**：阶段1 读取（OCR/抽取）、阶段2 生成，**各带一道界面内可编辑确认闸门**（简略展示、可改、可增条目）。
- **PDF 有文字版/图片版之分**：文字版抽文本层，图片版走视觉 OCR。
- **模型**：主 LLM 取自配置（当前 gpt-5.5，源码 `gpt-4o` 仅兜底）；OCR/视觉走独立 `VISION_*` 配置，**具体 OCR 模型待定、可插拔**。
