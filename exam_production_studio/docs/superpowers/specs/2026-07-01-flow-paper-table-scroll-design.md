# 流程页卷状态表格滚动设计

日期：2026-07-01

## 背景

流程执行页右侧「卷号 / 状态」表格会随着卷数增加继续撑高页面，导致任务概况区域被内容暴力拉长，页面可读性下降。

## 目标

让卷状态表格适应窗口高度：表格区域有最大高度，超出后在表格内部滚动，避免右侧栏无限向下延伸。

## 非目标

- 不改变后端数据结构。
- 不改变卷号和状态字段含义。
- 不改日志栏布局。
- 不引入新的 UI 依赖。

## 方案

只修改 `frontend/src/views/Flow.vue`：

1. 为右侧卷状态表格增加 `paperTableMaxHeight` 计算值。
2. 使用 Element Plus `el-table` 的 `:max-height` 属性，让表头固定、表体滚动。
3. 高度采用窗口自适应范围：最小 180px，最大 360px，默认按 `window.innerHeight - 460` 计算。
4. 监听窗口 resize，窗口大小变化时更新表格高度。
5. 页面卸载时移除 resize 监听，避免泄漏。

## 验证

- 前端构建通过。
- 在流程执行页查看右侧卷状态表格：卷数较多时表格内部出现滚动条，右侧按钮不会被大量卷记录挤到很远。
