# 中间产物排查增强 · 步骤文档

> 目标：解决两处排查短板——(1) 中间文件在界面上没入口、(2) LLM 输入输出无留档。
> 原则：每步只做一件事，改动尽量为「加法」不动现有流程；A 先做（它是 B 产出的查看器），B 后做。
> 相关既有代码：artifacts 接口 `backend/routers/artifacts.py`；工作目录 `04_生成输出`（`context.py:79-89`）；
> LLM 单一入口 `shared/ai/llm.py::call_api`；编排 `engine/runner.py`。

## 阶段 A：中间文件浏览页（单开新页 + 内联预览）

- [x] A1. 后端新增列目录树接口 `GET /api/projects/{id}/artifacts/tree?base=04_生成输出`：递归遍历，返回 `[{name, path(相对 ctx.root), size, is_dir, mtime, suffix}]`；用现有 `_rel()` 生成相对路径；做 `_within()` 越界校验。
- [x] A2. 后端加固现有 `download` 接口路径校验：把 startswith 前缀判断改为 `_within()`（relative_to），堵前缀绕过漏洞。
- [x] A3. 后端新增 xlsx 预览接口 `GET .../artifacts/preview-xlsx?path=`：用 openpyxl 读规划表/映射表，返回各 sheet 的二维数组（供前端表格渲染）。
- [x] A4. 前端 `api.ts` 增加 `artifactTree` / `previewXlsx` 方法（下载复用已有 `downloadUrl`）。
- [x] A5. 前端新增 `views/Files.vue` + 路由 `/projects/:id/files` + 侧栏「中间文件」入口；左侧 el-tree 目录树渲染 `04_生成输出`。
- [x] A6. 前端右侧内容预览按类型分流：`.json`→格式化；`.md/.txt/.log`→等宽文本；`.xlsx`→表格（调 A3）；图片→直接显示；`.pdf`→iframe；其它→下载。
- [x] A7. 前端对 `运行记录/state.json` 做特殊看板：展示已完成前置阶段 + 已完成卷号（一眼看卡在哪）。
- [x] A8. 联调：真实项目里在「中间文件」页逐类型验证浏览+预览通过（目录树 / state.json 看板 / 题目JSON / 规划表xlsx 等）。

## 阶段 B：LLM 输入输出落盘（默认开启 + 自动清理）

- [x] B1. 后端新增 `shared/ai/trace.py`：用 `contextvars.ContextVar` 存 `{dir, run_id, node, paper_no, seq}`；提供 begin/stage/end/log_call；序号自增。（已单测：begin→stage→log_call 正确落盘）
- [x] B2. 后端 `runner`：`_run` 开头 `trace.begin`；前置阶段 `trace.stage(label)`；逐卷 `trace.stage("拉题与补题"/"质检导出", pno)`；`_safe_run` finally `trace.end`。（调用点一行未改）
- [x] B3. 后端 `call_api` 内加落盘：读 trace，成功与异常都写 `运行记录/AI调用/<run_id>/{seq}_{node}_第N卷.json`，记录 时间/节点/卷号/model/temperature/max_tokens/messages/response(或error)/usage/耗时。
- [x] B4. 后端加设置项 `ai.trace_enabled`（默认 true）+ `ai.trace_keep_runs`（默认 10，begin 时清理旧运行目录）；config.get_ai_trace_config；设置接口加 `ai` 分组；全局设置页加「排查/调试」开关。
- [x] B5. 前端：AI调用目录由阶段 A 浏览页（目录树递归）直接查看，JSON 自动格式化，无需单独页面。
- [x] B6. 联调：真实运行一次，`运行记录/AI调用/<run_id>/` 正确落盘 4 条（补题/质检 × 第1/2卷），含完整 prompt、耗时、error。首跑即暴露真因：LLM 连接超时（APIConnectionError），导致题目全为占位符。
