# 目录架构说明

> 本文件用于帮助新用户快速理解当前文件夹 `wyy` 的用途、目录分工和常见使用流程。  
> 当前文件夹是一个“高职/高职分类考试一课一练试卷生成工具包”，核心目标是：根据考纲、教材、真题风格和考点规划表，批量生成一课一练试卷，并输出 Word、原始文本、质检报告等结果。

> 更新日期：2026-06-13。本文按当前目录实际结构更新，重点补充了已拆分的 `01_工具脚本/生成器/` 模块。

---

## 1. 顶层目录总览

```text
wyy/
├── create.py                         # 主入口脚本：调用 01_工具脚本/生成器/runner.py
├── generate_plan.py                  # 规划表生成入口：转发到 01_工具脚本/规划表/generate_plan.py
├── create_backup_20260613_before_split.py
│                                      # create.py 拆分模块前的备份文件
├── .token_usage.json                 # API token 使用记录
├── .claude/                          # Claude Code 本地配置/会话相关文件，一般不用手动处理
├── .workbuddy/                       # 历史记忆/辅助文件，一般不用手动处理
├── 01_工具脚本/                      # 各类辅助脚本，按功能分类存放
├── 02_配置资源/                      # 配置文件、题型定义、模板、编写规范
├── 03_项目数据/                      # 输入资料：考纲、教材、真题题库、OCR 文本、真题风格等
├── 04_生成输出/                      # 输出资料：考点规划表、生成结果、质检报告
├── 05_项目文档（使用前必读！）/                      # 项目说明文档
└── 目录架构说明.md                   # 本说明文件
```

顶层最重要的入口是：

- `create.py`：生成试卷的主入口。当前只负责设置路径并调用 `01_工具脚本/生成器/runner.py`。
- `generate_plan.py`：生成考点规划表的便捷入口，真实实现位于 `01_工具脚本/规划表/generate_plan.py`。
- `02_配置资源/config.json`：API、模型、输出目录等配置。
- `02_配置资源/编写规范/编写规范.md`：出题质量标准，是生成题目时最重要的规范文档。
- `04_生成输出/考点规划表/`：生成试卷前需要选择的规划表。
- `04_生成输出/生成结果/`：生成后的 Word、原始文本、压缩包等结果。

## 2. 各目录用途说明

## 2.1 `01_工具脚本/`：工具、模块与处理脚本

该目录存放项目运行过程中会用到的脚本，按功能拆分为多个子目录。

```text
01_工具脚本/
├── OCR/                              # 图片型 PDF OCR 处理工具
├── 文档处理/                         # Word/文档后处理脚本
├── 核心脚本/                         # Word 等核心工具库
├── 生成器/                           # 当前 create.py 调用的模块化生成器核心
├── 生成试卷/                         # 预留或扩展的试卷生成脚本目录
├── 真题风格/                         # 从真题中提取/蒸馏风格库
├── 规划表/                           # 生成和优化考点规划表
├── 质检/                             # 对生成试卷进行质量检查
├── 资料提取/                         # 从 PDF 等资料提取文本
└── 通用脚本/                         # 通用后处理、修复、打包等脚本
```

### `01_工具脚本/生成器/`

这是当前试卷生成流程的核心模块。`create.py` 已拆分为轻量入口，实际逻辑在该目录内。

常见文件：

- `runner.py`：一课一练生成主流程；解析命令行参数、选择规划表、循环生成、质检和保存。
- `config_io.py`：读取配置、读取编写规范、API 调用、token 使用记录。
- `paths.py`：项目根目录等路径常量。
- `planning.py`：解析考点规划表，生成输出路径基础信息。
- `prompts.py`：构建生成试卷使用的系统提示词和用户提示词。
- `text_generation.py`：与文本生成相关的处理。
- `text_processing.py`：清洗、规范化、拆分主题编号、提取题目摘要等文本处理。
- `references.py`：加载教材、真题和结构化真题风格库参考资料。
- `docx_generation.py`：调用 Word 生成逻辑，输出解析版/原卷版等文件。
- `postprocess.py`：生成后的后处理流程。

一般生成试卷时仍然运行根目录的 `create.py`，不需要直接运行 `生成器/` 内的模块。

### `01_工具脚本/OCR/`

用于处理扫描版、图片型 PDF。

常见文件：

- `ocr_pdf.py`：将图片型 PDF 转为文本、JSON、Markdown。
- `requirements.txt`：OCR 依赖。
- `README.md`：OCR 使用说明。
- `GLM-OCR部署.txt`：OCR 部署或使用记录。

适用场景：

- 真题 PDF 无法复制文字；
- 教材或试卷是扫描件；
- 需要先 OCR，再人工整理代表样题或目录文本。

示例：

```bash
cd 01_工具脚本/OCR
pip install -r requirements.txt
python ocr_pdf.py --pdf "../../03_项目数据/真题题库/重庆市/汽车类/某真题.pdf" --output-dir "output/某真题"
```

### `01_工具脚本/规划表/`

用于根据考纲、教材目录等生成“一课一练考点规划表”。

常见脚本：

- `generate_plan.py`：从考纲 PDF 生成考点规划表。
- `textbook_toc_scanner.py`：扫描教材目录，辅助按教材章节生成规划。
- `generate_plan_tables.py`、`optimize_themes.py`：规划表相关辅助处理。
- `example_usage.py`：示例用法。

适用场景：

- 新增省份、考类、教材；
- 需要根据考纲自动拆分练习主题；
- 需要按教材目录生成一课一练主题。

### `01_工具脚本/真题风格/`

用于从历年真题中提取命题风格。

常见脚本：

- `extract_exam_style.py`：根据真题样本、真题目录或题库，生成真题风格库。

适用场景：

- 希望生成题目更接近某省某考类真题口吻；
- 新增省份或考类时，需要先建立风格库；
- 真题只用于“风格迁移”，不得照搬题干、选项、情境或数值。

### `01_工具脚本/质检/`

用于检查生成试卷质量。

常见脚本：

- `check.py`：检查答案自暴露、题干重复、解析过短、选项失衡等问题。

适用场景：

- 生成后人工复核前先做自动质检；
- 检查已有 Word 或文本试卷是否存在明显问题。

### `01_工具脚本/资料提取/`

用于从教材、考纲、资料 PDF 中提取文本。

常见脚本：

- `extract_ref.py`：从文字型 PDF 提取参考资料文本。

### `01_工具脚本/核心脚本/`

核心工具库目录。

常见文件：

- `docx_utils1.py`：Word 文档生成和格式处理工具。

一般用户通常不需要直接运行该目录脚本，主要由 `01_工具脚本/生成器/docx_generation.py` 间接调用。

### `01_工具脚本/文档处理/`

用于处理已生成或已有的 Word 文档。

常见文件：

- `red_answers.py`：与答案标红或 Word 文档后处理相关的脚本。
- `batch_fix_math_docx.py`：批量修复 Word 中残留的 `{math:...}`/`{{math:...}}` 公式标记，转换为 Word 原生公式；同时可修复【答案】/【解析】段落正文整体加粗、答案前多余空白段落等历史格式问题。

示例：

```bash
python 01_工具脚本/文档处理/batch_fix_math_docx.py --path "04_生成输出/生成结果/重庆市 机械加工类"
python 01_工具脚本/文档处理/batch_fix_math_docx.py --dry-run
```

### `01_工具脚本/通用脚本/`

包含若干通用脚本，例如：

- `answer2none.py`：解析版转原卷版，移除答案与解析。
- `zip.py`：打包生成文件。
- `auto_fix_errors.py`：自动修复部分出题问题。
- `error_report.py`：生成或整理错误报告。
- `process_exam.py`、`generate_paper.py`：历史或通用处理脚本。
- `scrape_audit.py`、`_fix_options_table.py`：专项检查或修复脚本。
- `原卷版/`、`解析版/`：通用脚本处理过程中的分类输出或辅助目录。

## 2.2 `02_配置资源/`：配置、模板与规范

该目录存放项目运行所需的配置文件、题型规则、Word 模板和出题规范。

```text
02_配置资源/
├── config.json                       # API 与模型配置
├── 模板和资源/                       # Word 模板、分隔图、模板应用脚本
├── 编写规范/                         # 出题质量规范
└── 题型定义/                         # 各考类/学科的题型配置 JSON
```

### `02_配置资源/config.json`

用于配置 API 调用参数和输出目录。

常见字段：

- `api_key`：API 密钥。
- `base_url`：API 服务商地址。
- `model`：使用的模型名称。
- `max_tokens`：单次生成最大 token 数。
- `temperature`：生成随机性。
- `output_dir`：输出目录，通常为 `生成结果`。

注意：

- `api_key` 属于敏感信息，不要上传到公开仓库或发给无关人员。
- 如果更换模型或服务商，优先修改此文件。

### `02_配置资源/编写规范/编写规范.md`

这是出题质量的核心规范文件。

它规定了：

- 题型标题格式；
- 选择题选项长度和干扰项要求；
- 填空题空格格式；
- 判断题格式；
- 答案和解析格式；
- 自暴露检查规则；
- 质检和修复流程；
- 真题风格使用边界。

如果不运行脚本，只想让 AI 在对话中直接出题，也可以把这个文件发给 AI，再补充省份、教材、章节、主题、题型数量等信息。

### `02_配置资源/题型定义/`

存放各学科、考类的 JSON 配置。

当前常见文件包括：

- `index.json`：题型定义索引。
- `机械加工类.json`
- `汽车类.json`
- `重庆汽车类.json`
- `计算机网络基础.json`
- `计算机组装与维修.json`
- `植物保护技术.json`

新增学科或考类时，通常需要：

1. 在此目录新增一个对应的 `.json` 文件；
2. 在 `index.json` 中注册；
3. 确认生成脚本可以根据规划表匹配到该配置。

### `02_配置资源/模板和资源/`

用于 Word 输出格式。

常见文件：

- `template.docx`：Word 模板。
- `separator.png`：编写说明或分隔图资源。
- `_apply_template.py`：模板应用脚本。

如果生成的 Word 页眉、页脚、字体、版式需要调整，通常从这里入手。

## 2.3 `03_项目数据/`：输入资料库

该目录存放生成试卷所需的基础资料，是“输入数据区”。

```text
03_项目数据/
├── 参考资料/
│   ├── 教材/                         # 教材 PDF，按省份/考类管理
│   ├── 教材OCR/                      # 教材 OCR、目录 OCR、结构化目录等结果
│   ├── 真题/                         # 参考真题资料目录，按需存放
│   └── 真题风格/                     # 从真题样本中提炼出的风格文件
├── 真题题库/                         # 历年真题 PDF/DOCX 等原始题库
└── 考试说明/                         # 各省各考类考试说明/考纲 PDF
```

### `03_项目数据/考试说明/`

存放考试说明或考纲 PDF。

当前可见资料包括：

- 重庆市机械加工类考试说明；
- 重庆市汽车类考试说明；
- 重庆市电气技术类考试说明。

用途：

- 用于生成考点规划表；
- 用于确定考查范围和知识点边界。

### `03_项目数据/参考资料/教材/`

存放教材 PDF，按省份和考类分类。

示例结构：

```text
03_项目数据/参考资料/教材/
├── 重庆市/
│   ├── 机械加工类/
│   └── 电气技术类/
└── 内蒙古自治区/
    └── 机电类/
```

当前包含机械基础、机械制图、机械加工技术、电工电子、电气测量、电机与电气控制、电子技术基础与技能等教材资料。

用途：

- 提供知识准确性依据；
- 辅助生成教材目录驱动的考点规划表；
- 后续可通过 OCR 或文本提取转换为可注入的参考文本。

### `03_项目数据/参考资料/教材OCR/`

存放教材 OCR 或目录 OCR 结果。

常见内容：

- `pages/`：页面图片；
- `json/`：每页 OCR 结构化结果；
- `txt/`：每页纯文本；
- `combined.txt`、`combined.md`：合并文本；
- `toc_raw.txt`、`toc_structured.json`：目录原文和结构化目录；
- `toc_match_report.md`：目录匹配报告。

用途：

- 缓存教材目录识别结果；
- 避免重复 OCR；
- 便于人工检查目录识别是否准确。

### `03_项目数据/参考资料/真题/`

存放从原始真题题库中提取或 OCR 后沉淀出来的可复用真题文本汇总，按省份和考类分类。

示例结构：

```text
03_项目数据/参考资料/真题/
└── 内蒙古自治区/
    └── 机电类/
        └── 内蒙古自治区机电类_真题OCR汇总.txt
```

用途：

- 保存 OCR/文本提取后的真题汇总文本，作为后续人工复查和风格蒸馏的稳定入口；
- 与 `03_项目数据/真题题库/` 区分：题库目录放原始 PDF/DOCX/TXT，本目录放处理后的可读文本；
- 与 `04_生成输出/OCR结果/` 区分：OCR 结果目录放过程产物，本目录放可复用资料层。

要求：

- 使用 Tesseract 或其他 OCR 处理真题后，不能只把结果留在 `04_生成输出/OCR结果/`；
- 应同步写入本目录下的 `{省份}{考类}_真题OCR汇总.txt`；
- 同时刷新 `03_项目数据/参考资料/真题风格/{省份}/{考类}/_自动汇总样本.txt`，避免继续使用 PDF 文本层乱码。

### `03_项目数据/参考资料/真题风格/`

存放从真题样本中提炼出的风格文件。

通常按省份、考类管理，例如：

```text
03_项目数据/参考资料/真题风格/
├── 重庆市/
│   └── 机械加工类/
├── 通用/
└── _测试类/
```

常见文件：

- `风格总则.txt`
- `单选题风格.txt`
- `判断题风格.txt`
- `填空题风格.txt`
- `综合题风格.txt`
- `代表样题.txt`

用途：

- 让生成题目更像目标省份、目标考类的真题；
- 只模仿设问方式、选项结构、解析风格；
- 不作为知识依据，不允许照搬原题。

生成 `create.py` 运行时会优先读取这里的结构化风格库。如果当前省份/考类目录缺失或为空，生成器会在正式出题前检查 `03_项目数据/真题题库/{省份}/{考类}/`，并可提示从原始题库自动蒸馏风格库。

如果当前省份/考类没有对应真题题库，生成器会询问是否需要补充。若选择暂不补充，会在 `03_项目数据/真题题库/{省份}/{考类}/真题风格配置.json` 中写入 `config: false`；在改回 `true` 或删除该配置文件前，该考类不会读取真题，也不会读取该考类或通用真题风格库。

### `03_项目数据/真题题库/`

存放历年真题和解析版资料，按省份和考类分类。

示例：

```text
03_项目数据/真题题库/
├── 重庆市/
│   ├── 机械加工类/
│   ├── 汽车类/
│   └── 电子技术类/
└── 内蒙古自治区/
    └── ...
```

用途：

- 提供真题样本；
- 可用于蒸馏真题风格库；
- 可用于人工比对生成题目的命题风格。

## 2.4 `04_生成输出/`：生成结果与中间产物

该目录存放项目运行后产生的结果，是“输出区”。

```text
04_生成输出/
├── 考点规划表/                       # 用于生成试卷的 xlsx 规划表
├── 生成结果/                         # 已生成的一课一练试卷、原始文本、压缩包
└── 质检报告/                         # 自动质检输出的报告
```

### `04_生成输出/考点规划表/`

存放 `.xlsx` 格式的考点规划表。

常见结构：

```text
04_生成输出/考点规划表/
├── 重庆市/
│   ├── 机械加工类/
│   ├── 汽车类/
│   ├── 土建类/
│   └── 电气技术类/
└── 四川省/
    └── 汽车类/
```

规划表通常决定：

- 省份；
- 考类；
- 教材；
- 章节；
- 练习主题；
- 知识点范围；
- 题型和题量；
- 难度结构；
- 对应考纲标号。

使用 `create.py` 生成试卷时，一般先选择这里的规划表。

### `04_生成输出/生成结果/`

存放最终生成的试卷。

典型结构：

```text
04_生成输出/生成结果/
├── _原始文本/                        # 早期或未按省份/教材归档的原始文本
├── 重庆市 机械加工类/
│   ├── 机械基础/
│   │   ├── 解析版/
│   │   ├── 原卷版/
│   │   └── _原始文本/
│   ├── 机械制图/
│   └── 机械加工技术/
├── 重庆市 汽车类/
│   └── 计算机组装与维修/
└── 重庆市 电气技术类/
    ├── 电工技术基础与技能/
    ├── 电子技术基础与技能/
    └── 电机与电气控制技术/
```

常见输出文件：

- `解析版/*.docx`：包含答案和解析的教师版。
- `原卷版/*.docx`：去除答案和解析的学生版。
- `_原始文本/*.txt`：生成时的纯文本原稿。
- `*.zip`：解析版和原卷版配对打包结果。

### `04_生成输出/质检报告/`

存放自动质检报告。

用途：

- 查看题目是否有答案自暴露；
- 查看是否有重复题干；
- 查看选项长度是否失衡；
- 查看解析是否过短或无因果说明；
- 判断是否需要人工复核。

## 3. 常见使用流程

## 3.1 首次使用：生成已有规划表中的试卷

### 第一步：安装依赖

在当前文件夹下打开终端，安装基础依赖：

```bash
pip install openai openpyxl python-docx pdfplumber PyPDF2
```

如需处理图片型 PDF，再安装 OCR 相关依赖：

```bash
cd 01_工具脚本/OCR
pip install -r requirements.txt
```

### 第二步：检查 API 配置

打开：

```text
02_配置资源/config.json
```

确认以下字段可用：

- `api_key`
- `base_url`
- `model`

注意不要把含有真实密钥的配置文件公开分享。

### 第三步：运行主脚本

在当前文件夹根目录运行：

```bash
python create.py
```

然后按提示选择：

1. 考点规划表；
2. 生成范围，例如 `1-5`、`3,7,12` 或 `all`；
3. 是否继续生成下一批。

生成结果通常会出现在：

```text
04_生成输出/生成结果/
```

脚本默认会根据 `02_配置资源/config.json` 中的 `output_dir` 字段，将结果写入 `04_生成输出/生成结果/`。实际生成位置也会在终端中显示。

## 3.6 处理扫描版 PDF

## 4. 资料流转关系

可以把本项目理解为以下流程：

```text
03_项目数据/考试说明/考纲 PDF
        ↓
生成或填写考点规划表 xlsx
        ↓
教材 PDF / 教材 OCR / 真题风格库 / 编写规范 / 题型定义
        ↓
create.py → 01_工具脚本/生成器/runner.py 调用 API 生成试卷
        ↓
自动质检、答案分布调整与定向修复
        ↓
解析版 docx + 原卷版 docx + 原始文本 txt + zip
        ↓
人工抽查与交付
```

各目录在流程中的角色：

| 阶段 | 目录/文件 | 作用 |
|---|---|---|
| 准备考纲 | `03_项目数据/考试说明/` | 存放考试范围和要求 |
| 准备教材 | `03_项目数据/参考资料/教材/` | 提供知识依据 |
| 准备 OCR | `03_项目数据/参考资料/教材OCR/` | 存放教材 OCR 和目录识别结果 |
| 准备真题 | `03_项目数据/真题题库/` | 提供风格样本来源 |
| 生成风格 | `03_项目数据/参考资料/真题风格/` | 存放真题风格库 |
| 配置规则 | `02_配置资源/` | 存放 API、题型、模板、规范 |
| 生成规划 | `04_生成输出/考点规划表/` | 存放一课一练主题安排 |
| 生成试卷 | `create.py`、`01_工具脚本/生成器/` | 主入口与核心生成模块 |
| 查看结果 | `04_生成输出/生成结果/` | 存放最终试卷 |
| 查看质检 | `04_生成输出/质检报告/` | 存放质检结果 |

## 5. 哪些文件可以改，哪些文件慎改

## 5.1 经常需要修改的文件

| 文件/目录 | 修改场景 |
|---|---|
| `02_配置资源/config.json` | 更换 API、模型、输出目录 |
| `04_生成输出/考点规划表/**/*.xlsx` | 调整练习主题、题量、知识点范围 |
| `02_配置资源/题型定义/*.json` | 新增或调整题型配置 |

| `03_项目数据/参考资料/教材/` | 新增教材资料 |
| `03_项目数据/真题题库/` | 新增真题资料 |
| `05_项目文档（使用前必读！）/` | 补充项目说明、流程说明、规划表说明 |

## 5.2 一般不建议随意修改的文件

| 文件/目录 | 原因 |
|---|---|
| `create.py` | 主入口虽很短，但影响所有生成流程入口 |
| `01_工具脚本/生成器/runner.py` | 当前主生成流程，修改后可能影响全部生成流程 |
| `01_工具脚本/生成器/quality.py` | 自动质检和定向修复逻辑，修改后可能影响题目质量判断 |
| `01_工具脚本/生成器/prompts.py` | 生成提示词核心逻辑，修改后会影响所有试卷生成风格 |
| `01_工具脚本/核心脚本/docx_utils1.py` | Word 格式核心工具，修改后可能影响版式 |
| `02_配置资源/模板和资源/template.docx` | Word 模板，改动会影响所有导出文档 |
| `02_配置资源/编写规范/编写规范.md` | 出题质量核心标准，修改前应先确认规则意图 |
| `04_生成输出/生成结果/` | 已生成结果，删除前确认是否已交付或备份 |
| `create_backup_20260613_before_split.py` | 拆分前备份文件，除非确认不再需要，否则保留 |

## 7. 快速定位问题

| 问题 | 优先检查位置 |
|---|---|
| API 调用失败 | `02_配置资源/config.json`、`01_工具脚本/生成器/config_io.py` |
| 生成入口异常 | `create.py`、`01_工具脚本/生成器/runner.py` |
| 生成结果找不到 | `04_生成输出/生成结果/` 或终端输出路径 |
| 题量不对 | 对应的 `04_生成输出/考点规划表/**/*.xlsx` |
| 题型不对 | `02_配置资源/题型定义/` 和规划表“题型”列 |
| 题目质量不稳定 | `02_配置资源/编写规范/编写规范.md`、`01_工具脚本/生成器/prompts.py`、质检报告 |
| 自动质检误判或漏判 | `01_工具脚本/生成器/quality.py`、`01_工具脚本/质检/check.py` |
| Word 格式不对 | `02_配置资源/模板和资源/template.docx`、`01_工具脚本/核心脚本/docx_utils1.py`、`01_工具脚本/生成器/docx_generation.py` |
| 真题风格不像 | `03_项目数据/参考资料/真题风格/` |
| 教材知识不准 | `03_项目数据/参考资料/教材/` 或 `03_项目数据/参考资料/教材OCR/` |
| 扫描 PDF 读不出来 | `01_工具脚本/OCR/` |
| 规划表生成不符合预期 | `generate_plan.py`、`01_工具脚本/规划表/`、`05_项目文档（使用前必读！）/规划表编写说明.md` |

## 8. 使用建议

- 生成正式试卷前，先小范围生成 1～2 练测试配置是否正确。
- 新增省份或考类时，先整理好“考试说明、教材、真题、规划表、题型定义”五类资料。
- 真题风格库只用于模仿风格，不要照搬真题内容。
- 修改 `02_配置资源/config.json` 时注意保护 API 密钥。
- 交付前建议查看 `质检报告`，并人工抽查 Word 版格式。
- 如果批量生成数量较大，建议分批运行，便于发现问题后及时调整规划表或规范。
- 修改 `01_工具脚本/生成器/` 内模块前，建议先保留备份或只做小范围测试。
