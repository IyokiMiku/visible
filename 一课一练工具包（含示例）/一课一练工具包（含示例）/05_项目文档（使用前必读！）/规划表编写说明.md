# 考点规划表编写说明

> **最高优先级声明**：本文档是考点规划表生成的**最优先、唯一格式与流程标准**。无论使用本项目脚本还是其他 AI 生成规划表，都必须优先阅读并遵守本文档；若其他文档、脚本注释或历史说明与本文档冲突，以本文档为准。
>
> 本文档要求：有教材目录时，规划表生成必须优先采用“教材目录驱动”流程——先读取教材目录，将目录章节/条目确定为不同的试卷主题；再读取考纲，把与这些试卷主题相符的考纲条目填写为“考纲知识点”；级别可根据考纲知识点文本，并结合 AI 辅助判断。没有教材目录或教材不可用时，才退回到原来的“只针对考纲”的生成方法。

---

## 一、输入与输出

**输入**：考纲 PDF，以及可选的教材目录 PDF。

- 双击运行 `generate_plan.py` 时，程序依次询问：考纲 PDF 文件路径、规划表标题前缀、是否需要扫描教材目录 PDF。
- 若选择不扫描教材，则只根据考纲 PDF 文本生成规划表。
- 若选择扫描教材，则按标题前缀解析省份和考类，并优先查找 `03_项目数据/参考资料/教材/{省份}/{考类}/` 下的 PDF。例如标题前缀为 `重庆市机械加工类` 时，应查找 `03_项目数据/参考资料/教材/重庆市/机械加工类/` 目录下的教材 PDF。
- 教材扫描模式只 OCR 教材目录页，默认扫描前 12 页，可通过 `--toc-pages` 指定目录页范围。
- 教材 OCR 默认复用缓存 `toc_raw.txt`；如需强制重新 OCR，应使用 `--no-reuse-ocr`。
- 若考纲只对应一本教材/一门课程，则生成 1 个规划表。PDF 的全文文本可用 pdfplumber 或 PyPDF2 提取。

**输出**：`.xlsx` 规划表
- 可按教材/课程分文件，也可**一个 xlsx 内含多门课程**（每行第一列为“课程”）。多课程同表时，**序号按课程分组、进入新课程从 1 重置**（见 §2.7）。
- 文件名为 `{省份}{学科}_{教材名}_一课一练考点规划表.xlsx`，例如 `重庆市机械加工类_机械基础_一课一练考点规划表.xlsx`、`重庆市机械加工类_机械制图_一课一练考点规划表.xlsx`。
- 若同名文件存在，自动加 `_v2`、`_v3`。
- **输出目录**：规划表（连同映射表等生产规划产物）输出到 `桌面/输出结果/生产规划/一课一练/{省份}_{考类}/`（导出根可由全局设置 `output.export_dir` 或环境变量 `EXPORT_DIR` 覆盖）。

## 二、xlsx 整体布局

> **格式更新（2026-07）**：一课一练规划表改为**扁平列式**——层级从"行承载（合并单元格）"改为
> "**列承载**"，第一列即课程，每行一个考点、自带课程/单元/章，**数据区无跨列合并**（便于后续变量读取）。
> 仅顶部标题/配置/教材 3 行做 A:K 合并居中（给人看）。以下 §2.1–§2.4 均按新格式。

### 2.1 行列分配（扁平 11 列 A–K）

| 行号 | 内容 |
|------|------|
| 1 | 标题行（合并 A1:K1） |
| 2 | 配置行（合并 A2:K2）：题型、难度 |
| 3 | 信息行（合并 A3:K3）：参考教材 |
| 4 | 空行 |
| 5 | 列标题行（11 列 A–K） |
| 6+ | 数据行：每行一个考点，自带 课程/单元/章，**无合并** |

### 2.2 列定义（11 列，A–K）

| 列 | 列宽 | 内容 | 对齐 |
|----|------|------|------|
| A（课程） | 16 | 课程/教材名，每行填写（一个 xlsx 通常一门课程） | 左对齐 |
| B（单元） | 22 | 一级标题（单元/章），如"第1章 电路的基本概念" | 左对齐 |
| C（章） | 18 | 二级标题（章/节），如"一、电路基础知识" | 左对齐 |
| D（序号） | 6 | 每门课程内从 1 开始连续；极重要占 2 个连续号 | 居中 |
| E（考纲知识点） | 56 | 考纲原文的考点描述，完整保留；句尾不得额外添加句号，若原文抽取后末尾有 `。`、`.`、`．`，写入规划表前应去掉 | 左对齐，自动换行 |
| F（试卷主题） | 18 | 见 §四 | 左对齐 |
| G（级别） | 8 | "极重要"/"重要"/"标准" | 居中 |
| H（题型） | 26 | 如"单选15+多选9+判断6" | 左对齐 |
| I（难度） | 10 | 如"80:10:10" | 居中 |
| J（套数） | 6 | 固定为 1 | 居中 |
| K（考纲标号） | 16 | 如"课程1§3(2)" | 左对齐 |

### 2.3 样式规范

**字体**：全部使用"微软雅黑"。

| 元素 | 字号 | 加粗 | 颜色 | 底纹 |
|------|------|------|------|------|
| 标题行（第 1 行） | 12 | 是 | — | 无 |
| 配置/信息行（第 2-3 行） | 9 | 否 | — | 无 |
| 列标题行（第 5 行） | 10 | 是 | 白色 | #4472C4（蓝色） |
| 极重要考点行 | 9 | 否 | — | #FFD7D7（浅红） |
| 重要/标准考点行 | 9 | 否 | — | #FFF2CC（浅黄） |

**边框**：列标题行和所有考点数据行加全细线边框。
**合并**：仅第 1-3 行 A:K 合并居中；**数据区不合并**（单元/章作为列在每行重复填写）。

**对齐**：
- 标题行、配置行、信息行：水平居中、垂直居中
- 列标题行：水平居中、垂直居中
- 数据行：序号/级别/套数居中，其余左对齐；考纲知识点自动换行

### 2.4 数据行结构（扁平，每行一考点）

不再有"课程行/节标题行"合并行；课程/单元/章作为普通列在每行重复填写：

```
标题行（合并 A:K）
配置行（合并 A:K）
信息行（合并 A:K）
[空行]
列标题行：课程 | 单元 | 章 | 序号 | 考纲知识点 | 试卷主题 | 级别 | 题型 | 难度 | 套数 | 考纲标号
考点行：机械基础 | 第1章 … | 一、… | 1 | … | 机器组成      | 标准   | … | … | 1 | 课程1§1(1)
考点行：机械基础 | 第1章 … | 一、… | 2 | … | 力学性能（一）| 极重要 | … | … | 1 | 课程1§1(2)
考点行：机械基础 | 第1章 … | 一、… | 3 | … | 力学性能（二）| 极重要 | … | … | 1 | 课程1§1(2)
…（极重要占两个连续序号，主题带（一）（二）；同一 xlsx 内序号连续递增）
```

- 层级（单元/章 + C 列试卷主题）供 `create.py` 生成正文三行标题（见 §2.6）：单元→第X单元、章→第Z章、试卷主题→第W节。

### 2.6 规划表层级与试卷标题输出规则

规划表推荐使用三级标题；`create.py` 会按“一级标题 → 二级标题 → 试卷主题”的层级生成试卷标题。

#### 三级标题模板（推荐）

结构为：

```text
一级标题行
  二级标题行
    考点行
```

生成试卷标题时输出为：

```text
省份（考试类型）一课一练
《教材名》（出版社版·第几版） 第y练
第X单元 一级标题名称 第Z章 二级标题名称 第W节 试卷主题
```

其中：

- `一级标题行` 对应试卷标题中的 `第X单元 一级标题名称`。
- `二级标题行` 对应试卷标题中的 `第Z章 二级标题名称`。
- C 列“试卷主题”对应试卷标题中的 `第W节 试卷主题`。
- `第X单元`、`第Z章`、`第W节` 均使用中文大写序号，如 `第一单元`、`第一章`、`第一节`。
- `第y练` 使用 A 列练习序号，如 `第1练`、`第12练`。
- 第二行只放教材版本和 `第y练`，不放试卷主题。

示例：

```text
内蒙古自治区（对口招生）一课一练
《电子技术基础与技能》（高教版·第四版） 第1练
第一单元 二极管及其应用 第一章 二极管基础知识 第一节 二极管的结构与特性
```

#### 层级命名硬性要求

- 一级标题必须是教材或规划表的一级目录名称，例如 `第1章 二极管及其应用`、`第9章 数模转换与模数转换`。
- 一级标题中不得出现 `一、`、`二、`、`1.`、`1．`、`1、` 等二级或条目编号；这些编号只能用于二级标题或更低层级。
- 二级标题可以是普通文本（如 `二极管基础知识`），也可以带序号（如 `一、二极管基础知识`、`1．二极管基础知识`）。若未带序号，生成标题时会按其在同一一级标题下的出现顺序自动编号为 `第一章`、`第二章`。
- 若某个一级标题下没有二级标题，则生成标题时默认按 `第一章` 处理，试卷主题仍按 `第一节`、`第二节` 递增。
- 同一二级标题下的考点行按出现顺序生成 `第一节`、`第二节`、`第三节`，节名使用 C 列“试卷主题”。

#### 两级标题模板（兼容旧表）

结构为：

```text
课程行
  节标题行
    考点行
```

旧表仍可解析，但新规划表应优先按上述三级标题模板编写，避免标题层级含义不清。

### 2.7 序号重置规则（重要）

A 列“序号”不是全省/全学科的总序号，而是**当前课程内的一课一练编号（第X练）**。

- **按课程分组：进入新课程时序号重新从 `1` 开始，命名“第X练”也随之从头。**（扁平表以 A 列“课程”值区分课程。）
- 同一门课程内，序号连续递增。
- “极重要”考点拆成两行时，占用两个**连续**序号（在同一课程内），例如 `3`、`4`，主题分别加（一）（二）。
- 一个 xlsx **可包含多门课程**（每门课程序号各自从 1 重置）；也可按需每门课程单独一个 xlsx。两种方式序号规则一致（课程内从 1）。

示例（同一 xlsx 内多课程，序号按课程重置）：

```text
重庆市机械加工类_一课一练考点规划表.xlsx
  课程 机械基础： 第1练 / 第2练 / … / 第17练
  课程 机械制图： 第1练 / 第2练 / …      ← 进入新课程，序号与“第X练”重新从 1
```

**给 AI 的执行要求**：扁平表每行第一列填“课程”；同一门课程内序号从 `1` 连续递增，**遇到新课程立即把序号重置为 `1`**（命名同步从“第一练”重新开始）。极重要考点在其所属课程内占两个连续序号。

**与 create.py 配合时的注意事项**：当前 `create.py` 使用 A 列“序号”作为生成范围筛选条件。因此必须保证每个 xlsx 内的序号唯一且连续。多本教材分别生成多个 xlsx 后，调用 `create.py --file 某一本教材的规划表.xlsx --range 1-5` 时，只会生成该教材的第 1-5 练，不会误选其他教材。

## 三、PDF 文本解析规则

按行扫描 PDF 提取的全文文本，依次匹配以下模式。

### 3.1 定位起始位置

找到包含"考试内容及要求"的行，从下一行开始解析考点。该行之前的内容全部忽略。

### 3.2 解析规则表

| 匹配目标 | 正则模式 | 动作 |
|----------|---------|------|
| 课程标题 | `课程[一二三四五六七八九十\d]+[：:]\s*(.+)` | 创建新课程，清空当前节 |
| 节标题 | `(\d+)[．.、]\s*(.+)`（且第二组不以"了解/理解/掌握/熟悉/能/会/认识"开头） | 在当前课程下创建新节 |
| "绪论" | 独立一行出现 | 作为节标题 |
| 考点行 | `[（(]\s*(\d+)\s*[）)]\s*(.+)` | 加入当前节的考点列表 |
| 参考教材 | `\d+[．.]\s*《(.+?)》` 或出现"参考教材"字样 | 提取教材信息后停止解析 |
| "四、" | 行首 | 停止解析 |
| 页码标记 | `-+\s*\d+\s*(of\|/)\s*\d+\s*-+` | 跳过 |
| 纯数字行 | `^\d+$` | 跳过 |

### 3.3 认知层次判定

从考点文本开头判定认知层次：

| 文本开头 | 层次 |
|----------|------|
| 掌握 / 熟练掌握 | **掌握** |
| 熟悉 / 理解 | **理解** |
| 能 / 会 | **应用** |
| 其他（了解 / 认识等） | **了解** |

### 3.4 参考教材提取

从"参考教材"所在行往后扫描，匹配格式 `序号．《书名》xxx出版社, 年份年月份第版次版`。出版社名简化为缩写：

| 全称 | 缩写 |
|------|------|
| 高等教育出版社 | 高教 |
| 机械工业出版社 | 机工 |
| 人民邮电出版社 | 人邮 |
| 重庆大学出版社 | 重大 |

## 四、试卷主题规范

### 4.1 硬性标准

| 规则 | 要求 |
|------|------|
| 字数 | 核心部分 ≤10 汉字（不含（一）（二）后缀） |
| 语义 | 对知识点的浓缩总结，不是缩写 |
| 去动词 | 去掉"掌握/熟悉/了解/理解/能/会/认识" |
| 去后缀 | 去掉"概念/知识/方法/原理/定义" |
| 像标题 | 读起来像一节课的名称 |

### 4.2 正反例

| 知识点 | 正确主题 | 错误主题（及原因） |
|--------|----------|-------------------|
| 认识机器组成及各组成部分的作用 | 机器组成 | 认识机器组成（保留动词，与知识点雷同） |
| 掌握圆柱齿轮主要参数及几何尺寸的计算 | 齿轮参数计算 | 掌握齿轮参数（保留动词）；圆柱齿轮主要参数及几何尺寸计算（太长） |
| 了解铁碳合金相图及其应用 | 铁碳合金相图 | 了解铁碳合金相图（保留动词） |
| 掌握钢的热处理工艺 | 钢的热处理 | 钢的热处理工艺（"工艺"可去） |
| 了解碳素钢、合金钢和铸铁的分类、牌号、性能和应用 | 钢铁材料分类 | 碳素钢合金钢铸铁分类（太长，12字） |
| 了解常用金属材料的力学性能指标 | 金属力学性能 | 常用金属材料力学的性能指标（太长，11字） |

### 4.3 极重要考点

极重要考点生成两行，序号连续递增，主题末尾分别加（一）（二）。两行的知识点、级别、题型、难度、套数完全相同。

## 五、重要性判定

### 5.1 考点级别（决定是否拆分）

| 认知层次 | 级别 | 行数 |
|----------|------|------|
| 掌握 | **极重要** | 2 行（序号 N, N+1） |
| 理解 / 应用 | **重要** | 1 行 |
| 了解 / 其他 | **标准** | 1 行 |

### 5.2 考纲标号格式

`课程{课程序号}§{节序号}({考点序号})`

课程序号从 1 开始，节序号从 1 开始，考点序号从 1 开始。

## 六、标题行内容

### 6.1 第 1 行（标题）

`{省份}{学科}《一课一练》考点规划表 v1`

示例：`重庆市机械加工类《一课一练》考点规划表 v1`

### 6.2 第 2 行（配置）

`题型：{题型配置} | 难度：{难度配比}`

示例：`题型：单选5+填空3+综合2 | 难度：80:10:10`

### 6.3 第 3 行（教材）

每个规划表只允许填写**当前这个 xlsx 对应的一本教材**，格式为：

`参考教材：《{书名}》{出版社缩写}第{版次}版`

示例：

```text
参考教材：《机械基础》高教第3版
参考教材：《机械制图》高教第5版
参考教材：《机械加工技术》机工第2版
```

若考纲含多本教材，不要在同一个 xlsx 的第 3 行连续写多本书，例如禁止写成：

```text
参考教材：《机械基础》高教第3版、《机械制图》高教第5版、《机械加工技术》机工第2版
```

正确做法是拆成多个独立规划表：

```text
重庆市机械加工类_机械基础_一课一练考点规划表.xlsx
第 3 行：参考教材：《机械基础》高教第3版

重庆市机械加工类_机械制图_一课一练考点规划表.xlsx
第 3 行：参考教材：《机械制图》高教第5版

重庆市机械加工类_机械加工技术_一课一练考点规划表.xlsx
第 3 行：参考教材：《机械加工技术》机工第2版
```

**重要**：第 3 行必须保留书名、出版社缩写和版次，供 `create.py` 生成文件名中的 `《教材名》（出版社版·第几版）`。**试卷命名中的教材名称必须以考纲/规划表第 3 行的参考教材为准，即使本地没有对应教材 PDF、没有扫描教材目录，或实际生成时未加载教材正文，也不能因为缺少教材文件而省略教材名。**若确实未解析到考纲参考教材，才写为 `参考教材：待填写`，但这种情况下后续生成文件名可能缺少教材名，应尽量从考纲原文人工补齐后再运行 `create.py`。

## 七、完整生成流程

### 7.0 流程优先级总则（其他 AI 必须优先遵守）

生成规划表时，必须按以下优先级选择流程：

1. **有教材目录时，优先使用教材目录驱动流程**：先读取教材目录，把教材目录中的章、节、项目、任务或核心条目确定为不同的“试卷主题”。试卷主题应来自教材目录，不应先由考纲自由概括。
2. **再读取考纲并匹配主题**：读取考纲全文，将其中与某个试卷主题相符、相近或可覆盖该主题的考纲条目，设置为该行 B 列“考纲知识点”。B 列应尽量保留考纲原文，不把教材目录条目伪装成考纲知识点。
3. **级别判断**：D 列“级别”可通过考纲知识点文本中的要求词、覆盖范围、基础程度和考试价值判断；需要语义判断时，可以使用 AI 辅助，但最终仍应服从 §五 的级别规则。
4. **没有教材时才使用普通考纲模式**：如果确实没有教材目录、教材 PDF 不可用、目录 OCR 失败且无法人工补充目录，才按 §7.1 的只针对考纲生成方法：从考纲结构直接抽取考点并概括试卷主题。
5. **冲突处理**：若其他 AI、其他说明文件或脚本默认先读考纲再生成主题，应改为本文档规定的流程；本文档优先级最高。

### 7.1 普通考纲模式

1. 用 pdfplumber/PyPDF2 提取考纲 PDF 全文文本
2. 按 §三 解析文本，得到课程→节→考点 结构
3. 按 §五 判定每个考点的级别
4. 按 §四 为每个考点命名主题（≤8字精简总结）
5. 按 §二 的布局和样式写入 xlsx
6. 保存到 `04_生成输出/考点规划表/{省份}/{考类}/{省份}{考类}_一课一练考点规划表.xlsx`

### 7.2 教材目录 OCR 驱动模式

双击运行 `generate_plan.py` 并选择“需要扫描教材目录 PDF”时，执行以下流程：

1. 先解析考纲 PDF，得到考纲课程、章节、考点和参考教材信息。
2. 从标题前缀解析省份和考类。例如 `重庆市机械加工类` 解析为省份 `重庆市`、考类 `机械加工类`。
3. 优先查找 `03_项目数据/参考资料/教材/{省份}/{考类}/` 下的教材 PDF。例如：`03_项目数据/参考资料/教材/重庆市/机械加工类/*.pdf`。
4. 若该目录未找到 PDF，再兼容查找 `教材/{省份}/{考类}/`。
5. 对每本教材 PDF 的目录页进行 OCR，默认页码为前 12 页；OCR 文本缓存到 `03_项目数据/参考资料/教材OCR/{省份}/{考类}/{教材名}/toc_raw.txt`。当前优先使用 `textbook_toc_scanner.py` 的结构化目录扫描：PyMuPDF 高清渲染、Tesseract 中文 OCR、目录页检测、层级解析；若 `--ocr-engine auto` 且 Tesseract 不可用，则回退到 `OCR/ocr_pdf.py` 的 RapidOCR 流程。
6. 若缓存 `toc_raw.txt` / `toc_structured.json` 已存在且非空，默认复用缓存，不重新 OCR；命令行使用 `--no-reuse-ocr` 时才强制重新 OCR。
7. 结构化 OCR 会额外输出 `toc_raw_by_page.md`、`toc_structured.json`、`toc_detected_pages.json`、`toc.md`，供人工校对和后续复用。
8. 将 OCR 得到的教材目录解析为章节/目录条目，再与考纲知识点进行本地关键词匹配和可选 AI 语义匹配，并输出 `toc_match_report.json/md` 作为校对报告。
9. 按每本教材分别生成一个规划表，B 列写匹配到的考纲知识点；未精确匹配的目录项写为 `待补充：教材目录主题“...”未在考纲中精确匹配`，H 列写 `待人工确认`。
10. 输出到 `04_生成输出/考点规划表/{省份}/{考类}/`，文件名为 `{标题前缀}_{教材名}_一课一练考点规划表.xlsx`。若同名文件存在，自动加 `_v2`、`_v3`。

示例：标题前缀为 `重庆市机械加工类`，教材名为 `机械基础` 时，输出文件为：

```text
04_生成输出/考点规划表/重庆市/机械加工类/重庆市机械加工类_机械基础_一课一练考点规划表.xlsx
```

表格内部标题仍为：

```text
重庆市机械加工类《一课一练》考点规划表 v1
```

### 7.3 教材目录扫描工具使用说明

本节合并原 `README_扫描工具使用说明.md` 的内容，作为教材目录 OCR 驱动模式的配套说明。`textbook_toc_scanner.py` 用于自动扫描扫描版（图像型）PDF 教材，提取目录结构，并输出 Markdown 与结构化 JSON 结果，供 `generate_plan.py` 继续匹配考纲并生成规划表。

#### 7.3.2 单独扫描教材目录

基本用法：

```bash
# 扫描单本教材
python textbook_toc_scanner.py 电工技术.pdf

# 指定 tessdata 路径（如语言包不在系统默认目录）
python textbook_toc_scanner.py 电工技术.pdf --tessdata ./tessdata

# 扫描并保存为 Markdown 文件
python textbook_toc_scanner.py 电工技术.pdf --output 目录.md

# 批量扫描（输出到同一文件）
python textbook_toc_scanner.py 书1.pdf 书2.pdf 书3.pdf --output 三本教材目录.md
```

进阶参数：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--pages 2-20` | 扫描的页码范围（目录一般在前 20 页） | `2-20` |
| `--tessdata ./tessdata` | 指定 tessdata 目录 | 系统默认 |
| `--dpi 3.0` | 渲染分辨率倍率，越高越准但越慢 | `2.5` |
| `--output 结果.md` | 输出 Markdown 文件路径 | 打印到屏幕 |
| `--keep-images` | 保留渲染的中间 PNG 图像 | 否 |
| `--out-dir ./images` | 图像、OCR 缓存或结构化产物保存目录 | 临时目录 |
| `--render-only` | 仅渲染图像，不做 OCR（调试用） | 否 |
| `--layout auto\|single\|double` | 目录版面识别：自动/单栏/双栏 | `auto` |
| `--preprocess` | OCR 前进行灰度、对比度、二值化增强 | 否 |
| `--structured` | 输出 `toc_raw.txt`、`toc_structured.json` 等结构化产物 | 否 |
| `--no-reuse` | 不复用已有结构化 OCR 缓存 | 否 |
| `--verbose` | 显示详细日志 | 否 |

#### 7.3.3 与 `generate_plan.py` 融合使用

`generate_plan.py` 的教材目录 OCR 驱动模式会优先调用 `textbook_toc_scanner.py` 的结构化目录扫描能力：

```bash
python generate_plan.py ^
  --pdf "考纲.pdf" ^
  --title "重庆市电气技术类" ^
  --textbook-driven ^
  --toc-pages 2-20 ^
  --ocr-engine auto ^
  --toc-dpi 2.5 ^
  --tessdata "C:/path/to/tessdata"
```

融合流程：

1. `generate_plan.py` 解析考纲 PDF，得到课程、节、考点、参考教材。
2. 自动查找 `03_项目数据/参考资料/教材/{省份}/{考类}/` 下的教材 PDF。
3. 对每本教材调用 `textbook_toc_scanner.py` 生成结构化目录。
4. 优先读取 `toc_structured.json`，用教材目录顺序组织规划表。
5. 将教材目录项与考纲知识点进行本地关键词匹配和可选 AI 匹配。
6. 每本教材独立生成一个 xlsx 规划表。

每本教材 OCR 缓存目录会生成：

| 文件 | 用途 |
|------|------|
| `toc_raw.txt` | 目录页 OCR 原文，兼容旧流程 |
| `toc_raw_by_page.md` | 按 PDF 页码分块，便于人工校对 |
| `toc_structured.json` | 结构化目录，供规划表生成器直接读取 |
| `toc_detected_pages.json` | 每页目录检测得分和原因 |
| `toc.md` | 清洗后的 Markdown 目录 |
| `toc_match_report.json/md` | 目录项与考纲知识点匹配报告 |

#### 7.3.4 作为 Python 模块调用


#### 7.3.5 OCR 工作流程

```text
PDF 文件
   ↓
[PyMuPDF] 将每页渲染为 PNG 图像（默认 ×2.5 分辨率）
   ↓
[可选图片增强] 灰度、对比度、二值化
   ↓
[Tesseract OCR + chi_sim] 识别每页文字
   ↓
[目录页检测] 对每页打分，识别哪些页是目录
   - 关键词得分：目录/单元/第X章/X.X 格式等
   - 结构得分：行末有页码、短行比例高等
   - 反向惩罚：出现例题/解：等正文特征则减分
   ↓
[结构解析] 从目录页文字中识别层级
   - 一级：第X章、单元X、预备知识等
   - 二级：X.X 格式、实训项目等
   ↓
[结构化输出] Markdown / toc_structured.json / toc_raw_by_page.md
   ↓
[规划表生成] 目录项匹配考纲知识点，写入 xlsx
```

#### 7.3.6 已知限制与调试

| 问题 | 说明 |
|------|------|
| 扫描质量差 | OCR 识别率受扫描分辨率影响，可提高 `--dpi` 或开启 `--preprocess` |
| 双栏排版 | 可尝试 `--layout double` 或 `--toc-layout double` |
| 目录页检测误判 | 正文首页有时与目录页特征类似，可调整 `--pages` / `--toc-pages` |
| 简体/繁体混排 | 语言包默认为 `chi_sim`，繁体识别率较低 |

如果 OCR 结果不理想，先用 `--render-only` 查看渲染图像，确认扫描质量：

```bash
python textbook_toc_scanner.py 电工技术.pdf --render-only --out-dir ./debug_images
```

然后直接查看 `./debug_images/电工技术/page_005.png` 等文件。
