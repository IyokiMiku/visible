## 三、XLSX 规划表格式规范（核心产出）

### 3.1 总体结构

每个 xlsx 只包含**一个 sheet**，按以下固定行序排列：

| 行号 | 内容 | 说明 |
|------|------|------|
| 1 | 标题行 | 合并A~H列，例如"重庆市电气技术类《一课一练》考点规划表 v2" |
| 2 | 配置行 | 题型和难度，例如"题型：单选15+多选9+判断6 \| 难度：80:10:10" |
| 3 | 教材行 | 参考教材名称和版次 |
| 4 | 空行 | 留白 |
| 5 | 列标题行 | 8列固定标题 |
| 6 | 课程行 | 合并A~H列，例如"课程一：电工基础" |
| 7+ | 数据区 | 节标题行（合并）+ 考点数据行（8列） |

### 3.2 列定义（8列，A~H）

| 列 | 标题 | 宽度 | 内容说明 |
|----|------|------|---------|
| A | 序号 | 6 | 整数，从1开始依次递增，跨节连续编号 |
| B | 考纲知识点 | 60 | 完整考点描述文字，极重要考点两行文字相同 |
| C | 试卷主题 | 18 | ≤8字简短主题名，用于命名试卷。极重要考点拆为"(一)""(二)" |
| D | 级别 | 8 | 四选一：极重要 / 重要 / 标准 / 扩展 |
| E | 题型 | 28 | 题型分配，如"单选15+多选9+判断6" |
| F | 难度 | 10 | 难度配比，如"80:10:10" |
| G | 套数 | 6 | 该主题对应试卷套数，通常为1 |
| H | 考纲标号 | 16 | 溯源编号，如"课程1§1(3)"，扩展填"扩展" |

### 3.3 级别含义与行底色

| 级别 | 考纲对应 | 含义 | 行底色 |
|------|---------|------|--------|
| 极重要 | "掌握" | 核心计算/分析型考点 | 浅红（#FFD7D7） |
| 重要 | "理解" | 原理理解型考点 | 浅黄（#FFF2CC） |
| 标准 | "了解" | 识记了解型考点 | 白色 |
| 扩展 | 无匹配 | 教材有但考纲未覆盖 | 浅灰（#EBEBEB） |

### 3.4 极重要考点拆分规则

"极重要"级别的考点**自动拆为2行**：
- 知识描述（B列）相同
- 主题名（C列）后缀加 "（一）""（二）"
- 序号（A列）独立编号
- 目的：为同一考点生成两套不同试卷，加深练习

### 3.5 章节组织原则

- **以教材目录为框架**：教材的每一章/每一个单元都必须在规划表中出现
- 就算考纲没有对应考点，教材的章节也**不能跳过**，以"扩展"级别保留
- 不需要理会教材中的"拓宽知识""相关技能""知识与技能考核""实训项目""应知应会""练习与考工模拟"等非主干小节

### 3.6 样式标准

- 字体：微软雅黑
- 标题行字号12、加粗
- 列标题：白字蓝底（#4472C4）、字号10加粗
- 课程行：浅蓝底（#D6E4F0）、字号10加粗
- 数据行：字号9，自动换行
- 全表统一细线边框

## 四、教材目录提取方法

### 4.1 问题

三本教材均为**扫描图像型 PDF**，无嵌入文本层、无书签目录。`PyMuPDF.get_toc()` 返回空列表，`get_text()` 仅能提取"扫描全能王"水印文字。

### 4.2 解决方案：PyMuPDF 渲染 + Tesseract OCR

```
PDF 文件
   ↓ [PyMuPDF] 逐页渲染为 PNG 图像（2.5x~3x 分辨率）
PNG 图像目录
   ↓ [Tesseract OCR + chi_sim 中文语言包]
OCR 文字文本
   ↓ [目录页检测 + 层级解析]
结构化 Markdown 目录
```

### 4.3 已封装的工具：`textbook_toc_scanner.py`

**位置**：项目根目录下的 `textbook_toc_scanner.py`

**功能**：
- PDF 页面渲染（PyMuPDF，可调分辨率）
- Tesseract OCR 识别（自动查找 tessdata，支持自定义路径）
- 目录页智能检测（关键词打分 + 行末页码 + 反正文惩罚）
- 层级结构解析（一级"第X章""单元X"、二级"X.X"格式）
- Markdown 格式输出
- 支持批量和单本两种模式
- 可作为命令行工具或 Python 模块 import 调用

**环境要求**：
```bash
pip install pymupdf pytesseract Pillow
# 安装 Tesseract OCR（Windows）：
winget install --id UB-Mannheim.TesseractOCR
# 中文语言包 chi_sim.traineddata 需放入 tessdata 目录
```

**命令行用法**：
```bash
# 扫描单本
python textbook_toc_scanner.py 电工技术.pdf --tessdata ./tessdata --output 目录.md

# 批量扫描
python textbook_toc_scanner.py 书1.pdf 书2.pdf 书3.pdf --tessdata ./tessdata --output 汇总.md

# 调高分辨率提升识别率
python textbook_toc_scanner.py 电工技术.pdf --dpi 3.0

# 仅渲染图像调试
python textbook_toc_scanner.py 电工技术.pdf --render-only --out-dir ./debug
```

**Python 模块调用**：
```python
from textbook_toc_scanner import scan_single, scan_multiple

# 扫描单本
md = scan_single("电工技术.pdf", tessdata_dir="./tessdata", dpi=2.5)

# 批量扫描
md = scan_multiple(["书1.pdf", "书2.pdf"], tessdata_dir="./tessdata", output="汇总.md")
```

### 4.5 已知限制

| 问题 | 影响 | 应对 |
|------|------|------|
| 扫描质量低 | OCR 部分字词识别错误 | 人工校对关键词；提高 --dpi |
| 双栏排版 | 两栏文字混合输出 | 酌情分段阅读 |
| 缺少中文语言包 | Tesseract 无法识别中文 | 需手动下载 chi_sim.traineddata |
| 目录页检测误判 | 正文首页可能被误判为目录 | 缩小 --pages 范围到前20页 |

## 五、规划表生成方法

### 5.1 已封装的工具：`generate_plan_v2.py`

**位置**：项目根目录下的 `generate_plan_v2.py`

**输入**：
- 硬编码在脚本中的三本教材结构化数据（教材章节 + 考纲匹配结果）
- 数据来源：教材目录（OCR提取 + 人工校对） + 考纲 PDF（PyMuPDF 文本提取）

**输出**：
- 三个 xlsx 文件，严格按第三节格式规范生成

**核心设计逻辑**：
1. 遍历教材每个章节，每个小节（sub-section）生成至少1行
2. 查找考纲中同级知识点，匹配则写入考纲标号
3. 无匹配的章节标为"扩展"级别
4. "极重要"（掌握级）知识点自动拆分2行

**重新生成**：
```bash
python generate_plan_v2.py
```

**修改数据**：编辑脚本中的 `DG_DATA`（电工）、`DZ_DATA`（电子）、`DJ_DATA`（电机）三个字典。

### 5.2 考纲知识点到教材章节的匹配逻辑

以《电工技术基础与技能》为例：

| 考纲 | 教材章节 |
|------|---------|
| 课程1§1 直流电路基本概念 | 第1章 直流电路基础 |
| 课程1§2 电阻电路分析 | 第2章 简单直流电路 + 第3章 复杂直流电路 |
| 课程1§3 磁场与电磁感应 | 第5章 磁场和磁路 + 第6章 电磁感应 + 第4章 电容 |
| 课程1§4 单相正弦交流电路 | 第7章 + 第8章 + 第9章 |
| 课程1§5 三相交流电路 | 第10章 三相正弦交流电路 |

注意：教材章节顺序与考纲单元顺序不完全一致（如教材第4章电容在磁场之前），规划表以**教材物理顺序**为准。
