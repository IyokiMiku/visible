# 一课一练试卷生成工具包

> 本项目是一个高职/高职分类考试“一课一练”试卷生产工具包。它围绕**考纲、教材目录、教材资料、真题风格、题型定义、编写规范、考点规划表和质检反馈**，批量生成解析版/原卷版 Word 试卷、原始文本、压缩包和质检报告。
>
> **给用户和其他 AI 的最高优先级说明**：本项目的规划表生成必须优先参考 `05_项目文档（使用前必读！）/规划表编写说明.md`。该文件是规划表生成的最优先、唯一格式与流程标准；若其他文档、脚本注释或历史说明与它冲突，以 `规划表编写说明.md` 为准。

---

## 1. 项目定位与个性化方法

本项目不是简单的“把考点丢给 AI 出题”，而是一套分层控制的试卷生产流程：

```text
教材目录决定一课一练主题
考纲决定知识点边界和考试要求
规划表决定每练题型、题量、难度和套数
编写规范决定通用质量底线
题型定义 JSON 决定当前考类的个性化题型画像
真题风格库决定设问口吻、选项结构、解析风格
教材/参考资料决定知识依据
质检与修复决定最终交付质量
```

核心原则：

1. **教材目录优先定主题**：有教材目录时，先读取教材目录，将章、节、项目、任务或核心目录条目确定为不同“试卷主题”。
2. **考纲匹配知识点**：再读取考纲，把与试卷主题相符的考纲条目设置为 B 列“考纲知识点”。B 列应尽量保留考纲原文，不把教材目录条目伪装成考纲知识点。
3. **没有教材才退回考纲模式**：没有教材目录、教材不可用或目录 OCR 失败且无法人工补充时，才使用旧的“只针对考纲”的规划表生成方法。
4. **规划表最高控制出题结构**：最终生成试卷时，规划表中的题型、题量、难度、主题和知识点优先级最高。
5. **真题只迁移风格，不迁移内容**：真题风格库只用于模仿设问口吻、选项结构、题型习惯和解析简洁程度，不作为知识依据，不得照搬真题题干、选项、情境或数值。
6. **AI 辅助但不替代规则**：AI 可辅助判断考纲知识点级别、主题匹配和风格总结，但必须服从规划表、编写规范和本文档体系。

## 2. 完整试卷生产流程

### 2.1 总流程

```text
准备资料
  ├─ 考试说明/考纲 PDF
  ├─ 教材 PDF 或教材目录 OCR 文本
  ├─ 真题 PDF/DOCX/TXT
  └─ 编写规范、题型定义、Word 模板
        ↓
生成或校正真题风格库
        ↓
生成考点规划表 xlsx
  ├─ 有教材：教材目录 → 试卷主题 → 匹配考纲知识点 → 判断级别
  └─ 无教材：考纲结构 → 考点 → 概括主题 → 判断级别
        ↓
运行 create.py 选择规划表和范围
        ↓
加载规划表 + 教材资料 + 真题风格库 + 题型定义 + 编写规范
        ↓
调用 API 生成试卷原始文本
        ↓
本地质检、答案分布调整、定向修复
        ↓
生成解析版 Word、原卷版 Word、原始文本和 zip
        ↓
人工抽查与交付
```

### 2.2 规划表阶段

规划表是试卷生产的核心控制文件，默认存放在：

```text
04_生成输出/考点规划表/{省份}/{考类}/
```

规划表生成标准见：

```text
05_项目文档（使用前必读！）/规划表编写说明.md
```

当前明确的规划表生成优先级：

1. **优先教材目录驱动**：读取教材目录后，将目录条目定为不同试卷主题。
2. **再读考纲**：将与主题相符的考纲内容写入“考纲知识点”。
3. **级别判断**：根据考纲文本中的要求词、覆盖范围、基础程度和考试价值判断；必要时用 AI 辅助。
4. **无教材退回旧流程**：只根据考纲 PDF 文本生成规划表。
5. **多教材拆分**：若考纲对应多本教材/多门课程，每本教材必须单独生成一个 xlsx，序号从 1 重新开始。

### 2.3 真题风格库阶段

真题风格库默认存放在：

```text
03_项目数据/参考资料/真题风格/{省份}/{考类}/
```

它通常包含：

```text
风格总则.txt
单选题风格.txt
多选题风格.txt
判断题风格.txt
填空题风格.txt
简答题风格.txt
计算题风格.txt
综合题风格.txt
代表样题.txt
_自动汇总样本.txt
```

近期已改动：`01_工具脚本/真题风格/extract_exam_style.py` 已支持在自动汇总真题样本时启用 Tesseract OCR 兜底。普通 PDF 文本层为空或出现 `{#{QQ...}#}` 等疑似乱码时，可加 `--ocr-pdf` 自动渲染 PDF 页面并用 Tesseract 识别。

### 2.4 试卷生成阶段

主入口：

```bash
python create.py
```

生成器会读取规划表中的：

- 一级标题行：生成正文标题中的“第X单元 一级标题名称”，一级标题不得含 `一、`、`1.`、`1．` 等二级或条目编号；
- 二级标题行：生成正文标题中的“第Z章 二级标题名称”，无二级标题时默认 `第一章`；
- B 列：考纲知识点；
- C 列：试卷主题，生成正文标题中的“第W节 试卷主题”；
- D 列：级别；
- E 列：题型与题量；
- F 列：难度比例；
- G 列：套数；
- H 列：考纲标号。

然后结合教材、真题风格库、题型定义、编写规范构建 prompt，生成每一练试卷。

### 2.5 质检与交付阶段

生成后会输出：

```text
04_生成输出/生成结果/
└── {省份} {考类}/
    └── {教材名}/
        ├── 解析版/
        ├── 原卷版/
        └── _原始文本/
```

质检关注：

- 题量是否符合规划表；
- 题型是否符合要求；
- 答案是否自暴露；
- 选项长度是否明显失衡；
- 答案分布是否过度集中；
- 解析是否过短或空泛；
- Word 格式、公式、答案/解析样式是否正常。

## 3. 快速开始

### 3.1 安装基础依赖

在项目根目录 `wyy/` 下执行：

```bash
pip install openai openpyxl python-docx pdfplumber PyPDF2 lxml latex2mathml
```

如需处理扫描版或图片型 PDF，再安装 OCR 依赖：

```bash
cd 01_工具脚本/OCR
pip install -r requirements.txt
```

如需使用 Tesseract OCR：

```bash
pip install pytesseract Pillow
winget install --id UB-Mannheim.TesseractOCR
```

中文语言包 `chi_sim.traineddata` 需要位于 Tesseract 的 `tessdata` 目录，或运行脚本时用 `--tessdata` 指定。

### 3.2 配置 API

编辑：

```text
02_配置资源/config.json
```

重点检查：

- `api_key`：API 密钥；
- `base_url`：API 服务商地址；
- `model`：模型名称；
- `output_dir`：输出目录名，默认配合 `04_生成输出/生成结果/` 使用。

> 注意：`api_key` 是敏感信息，不要上传到公开仓库或发给无关人员。

### 3.3 生成试卷

```bash
python create.py
```

脚本会从以下目录递归列出可用规划表：

```text
04_生成输出/考点规划表/
```

按提示选择规划表和生成范围即可。

## 5. 代码模块说明

### 5.1 生成器模块：`01_工具脚本/生成器/`

`create.py` 目前是轻量入口，核心逻辑集中在：

```text
01_工具脚本/生成器/
```

| 模块 | 作用 |
|---|---|
| `runner.py` | 主流程：解析参数、选择规划表、生成、质检、保存和后处理 |
| `config_io.py` | 配置读取、API 调用、token 用量统计 |
| `planning.py` | 解析考点规划表，确定省份、考类、教材、主题和输出路径 |
| `prompts.py` | 构建生成试卷所用提示词 |
| `references.py` | 加载教材、真题和结构化真题风格库参考资料 |
| `text_generation.py` | 文本生成相关辅助处理 |
| `text_processing.py` | 文本清洗、规范化、主题编号拆分、题目摘要提取 |
| `quality.py` | 本地质检、答案分布调整、定向修复和质检摘要 |
| `docx_generation.py` | 生成解析版 Word，并调用核心 Word 工具 |
| `postprocess.py` | 生成原卷版、打包 zip、结果分类等后处理 |
| `paths.py` | 项目根目录和关键路径常量 |

一般使用时仍运行根目录 `create.py`；修改生成逻辑时优先定位到上述模块。

### 5.2 规划表模块：`01_工具脚本/规划表/`

| 文件 | 作用 |
|---|---|
| `generate_plan.py` | 从考纲 PDF、教材目录等生成考点规划表 |
| `textbook_toc_scanner.py` | 用 PyMuPDF + Tesseract 扫描教材目录，输出结构化目录 |
| `generate_plan_tables.py` | 规划表辅助生成 |
| `optimize_themes.py` | 主题优化辅助 |
| `generate_neimenggu_plan.py` | 内蒙古机电类无考纲模式规划表生成脚本 |

规划表规则优先看：

```text
05_项目文档（使用前必读！）/规划表编写说明.md
```

### 5.3 真题风格模块：`01_工具脚本/真题风格/`

| 文件 | 作用 |
|---|---|
| `extract_exam_style.py` | 从真题样本、真题目录或题库生成真题风格库 |

近期改动：

- 支持 `--ocr-pdf`：对空文本层或乱码 PDF 启用 Tesseract OCR；
- 支持 `--ocr-pages`：只 OCR 指定页；
- 支持 `--ocr-dpi`：调整 PDF 渲染倍率；
- 支持 `--ocr-lang`、`--tessdata`：指定语言和语言包目录；
- 支持 `--ocr-preprocess`：OCR 前灰度/对比度/二值化增强；
- 支持 `--sample-only`：只生成 `_自动汇总样本.txt`，不生成风格模板且不调用 API。

### 5.4 OCR 模块：`01_工具脚本/OCR/`

| 文件 | 作用 |
|---|---|
| `ocr_pdf.py` | 将图片型 PDF 转为页面图片、JSON、TXT、Markdown |
| `requirements.txt` | RapidOCR OCR 依赖 |
| `README.md` | OCR 方案说明 |

当前 OCR 方式：

- 真题风格自动汇总可用 **Tesseract OCR 兜底**；
- 独立图片型 PDF OCR 工具使用 **PyMuPDF + RapidOCR**；
- 教材目录结构化扫描优先使用 **PyMuPDF + Tesseract**。

### 5.5 质检模块：`01_工具脚本/质检/`

| 文件 | 作用 |
|---|---|
| `check.py` | 检查答案自暴露、题干重复、解析过短、选项失衡等问题 |

### 5.6 文档处理与 Word 生成

| 位置 | 作用 |
|---|---|
| `01_工具脚本/核心脚本/docx_utils1.py` | Word 生成核心工具 |
| `01_工具脚本/文档处理/batch_fix_math_docx.py` | 批量修复公式、答案/解析样式 |
| `02_配置资源/模板和资源/template.docx` | Word 模板 |

## 6. 常用命令

### 6.1 生成试卷

```bash
# 交互式选择规划表和范围
python create.py

# 指定规划表和范围
python create.py --file "04_生成输出/考点规划表/重庆市/机械加工类/重庆市机械加工类_机械基础_一课一练考点规划表.xlsx" --range 1-5

# 生成全部；all 模式会按每批 5 个主题生成，批次间暂停 20 秒，降低连续 API 调用出错概率
python create.py --file "04_生成输出/考点规划表/重庆市/机械加工类/重庆市机械加工类_机械基础_一课一练考点规划表.xlsx" --range all

# 跳过自动质检
python create.py --file "04_生成输出/考点规划表/重庆市/机械加工类/重庆市机械加工类_机械基础_一课一练考点规划表.xlsx" --range 1-3 --no-check
```

### 6.2 生成考点规划表

根目录便捷入口：

```bash
python generate_plan.py --pdf "03_项目数据/考试说明/某考试说明.pdf" --title "重庆市机械加工类"
```

教材目录驱动模式示例：

```bash
python generate_plan.py --textbook-driven \
  --pdf "03_项目数据/考试说明/2025 年重庆市高等职业教育分类考试专业综合理论测试 机械加工类考试说明1.pdf" \
  --title "重庆市机械加工类" \
  --textbook-dir "03_项目数据/参考资料/教材/重庆市/机械加工类" \
  --toc-pages 1-12
```

OCR 缓存默认写入：

```text
03_项目数据/参考资料/教材OCR/{省份}/{考类}/{教材名}/
```

```bash
python 01_工具脚本/质检/check.py --file "04_生成输出/生成结果/某试卷.docx"

```

## 7. 新增省份或考类的推荐流程

1. 将考试说明/考纲 PDF 放入：

   ```text
   03_项目数据/考试说明/
   ```

2. 将教材 PDF 放入：

   ```text
   03_项目数据/参考资料/教材/{省份}/{考类}/
   ```

3. 将真题 PDF/DOCX/TXT 放入：

   ```text
   03_项目数据/真题题库/{省份}/{考类}/
   ```

4. 使用真题风格脚本生成或更新：

   ```text
   03_项目数据/参考资料/真题风格/{省份}/{考类}/
   ```

5. 按 `05_项目文档（使用前必读！）/规划表编写说明.md` 生成规划表：
   - 有教材：先教材目录定主题，再匹配考纲知识点；
   - 无教材：按考纲结构生成主题和知识点。

6. 如该考类没有题型定义，在 `02_配置资源/题型定义/` 中新增 JSON，并在 `index.json` 中注册。

7. 运行：

   ```bash
   python create.py
   ```

8. 查看质检报告并人工抽查 Word 文件。

## 8. 质量标准与配置优先级

### 8.1 质量标准

详见：

```text
02_配置资源/编写规范/编写规范.md
```

核心规则包括：

- 选择题选项长度尽量均衡；
- 禁止答案关键词在题干中自暴露；
- 填空题用 `______`；
- 判断题答案使用 `√` 或 `×`；
- 解析必须是完整因果句；
- 真题风格只模仿风格，不照搬内容；
- 主观题难度标签遵循用户偏好：使用“简单/适中/困难”，3 道及以上主观题时约 1/3 为“困难”，同一大题内按简单→适中→困难排序。

### 8.2 生成时的优先级

```text
1. 规划表题型、题量、主题、知识点、难度、套数
2. 当前教材、考纲和本主题知识依据
3. 编写规范.md
4. 题型定义 JSON
5. 当前省份/考类真题风格库
6. 通用真题风格库
7. AI 自身常识
```

说明：

- 规划表决定最终题型和题量。
- 教材和考纲决定知识正确性。
- 编写规范决定质量底线。
- 题型定义 JSON 提供当前考类的个性化题型画像。
- 真题风格库只决定“像不像”，不决定“对不对”。

## 9. 已声明的重要改动

截至当前，本 README 明确声明以下项目改动和约定：

1. **规划表生成说明最高优先级**  
   `05_项目文档（使用前必读！）/规划表编写说明.md` 是规划表生成的最优先说明，其他 AI 生成规划表时必须优先参考。

2. **教材目录优先的规划表流程**  
   有教材目录时，先读取教材目录并确定试卷主题，再读取考纲匹配考纲知识点；没有教材时才退回只针对考纲的生成方法。

3. **级别可由 AI 辅助判断**  
   D 列“级别”可根据考纲知识点文本和 AI 语义辅助判断，但必须服从规划表说明中的规则。

4. **真题风格自动汇总支持 Tesseract OCR 兜底**  
   `extract_exam_style.py` 可通过 `--ocr-pdf` 对空文本层或乱码 PDF 进行 Tesseract OCR，并生成对应 `_自动汇总样本.txt`。

5. **生成器模块化**  
   `create.py` 已拆分为轻量入口，核心逻辑位于 `01_工具脚本/生成器/`。

6. **多教材必须拆分规划表**  
   多本教材/多门课程不得混在一个 xlsx 中；每本教材单独一个规划表，序号从 1 开始。

7. **真题风格不得照搬**  
   真题只用于风格蒸馏和结构参考，不作为知识依据，不得复制原题。

8. **试卷命名以考纲参考教材为准**  
   试卷文件名和 Word 标题中的 `《教材名》（出版社版·第几版）` 必须来自规划表第 3 行的考纲参考教材；即使本地没有对应教材 PDF、未扫描教材目录或未加载教材正文，也不能省略教材名。

## 10. 常见问题定位

| 问题 | 优先检查 |
|---|---|
| API 调用失败 | `02_配置资源/config.json` |
| 找不到规划表 | `04_生成输出/考点规划表/` |
| 不确定规划表怎么写 | `05_项目文档（使用前必读！）/规划表编写说明.md` |
| 生成结果找不到 | `04_生成输出/生成结果/` 或终端输出路径 |
| 题量不对 | 规划表中的“题型”列 |
| 题型不匹配 | `02_配置资源/题型定义/` 和规划表 E 列 |
| 题目质量不稳定 | `02_配置资源/编写规范/编写规范.md` 和质检报告 |
| Word 格式不对 | `02_配置资源/模板和资源/template.docx`、`01_工具脚本/核心脚本/docx_utils1.py` |
| 真题风格不像 | `03_项目数据/参考资料/真题风格/` |
| 真题 PDF 是乱码 | `extract_exam_style.py --ocr-pdf` 或 `01_工具脚本/OCR/ocr_pdf.py` |
| 教材知识不准 | `03_项目数据/参考资料/教材/` 或教材 OCR 缓存 |
| 教材目录识别不准 | `03_项目数据/参考资料/教材OCR/`、`textbook_toc_scanner.py`、`--toc-pages`、`--tessdata` |

## 11. 注意事项

- `04_生成输出/生成结果/` 是结果区，不应放脚本副本；工具脚本统一放在 `01_工具脚本/`。
- 修改 `create.py`、`01_工具脚本/生成器/`、`01_工具脚本/核心脚本/docx_utils1.py` 前要谨慎，因为会影响批量生成流程和 Word 格式。
- 修改规划表生成逻辑前，必须先阅读 `05_项目文档（使用前必读！）/规划表编写说明.md`。
- 交付前建议先查看质检报告，再人工抽查 Word 文件。
- 批量生成建议分批运行，便于发现问题后及时调整规划表、教材资料或编写规范。
- OCR 结果必须人工抽查；扫描件识别错误不应直接作为知识依据。
