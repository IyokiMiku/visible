## 二、生成规范（由 AI 执行时遵守）

### 2.1 批次控制

| 规则 | 说明 |
|------|------|
| 每批上限 | ≤ 3 卷 |
| 原因 | bash 超时限制（默认10分钟），大批量会导致批次后半截丢失 |
| 严禁 | 一次性跑 `--range 1-50` 或 `for 循环内大量批次` |

### 2.2 生成流程（强制性规范）

1. 从卷数最少的科目开始（验证流程）
2. 每批 ≤ 3 卷，等待返回后检查分数，确认无误再跑下一批
3. **严禁大批量**：不要把几十卷塞进一个 for 循环。API 调用中断或 bash 超时会导致后续批次全部丢失，已生成的文件虽然还在，但断点续跑和查缺非常耗时。
4. **每完成一本教材，立即查漏补缺**：一门科目全部跑完后，**必须**执行下方检查脚本，确认 0 缺失后才开始下一本教材。

```python
import os, re
def check_gaps(dir, expected):
    found = set()
    for f in os.listdir(dir):
        m = re.match(r'第(\d+)练', f)
        if m: found.add(int(m.group(1)))
    missing = sorted(set(range(1, expected+1)) - found)
    if missing:
        print(f'缺失 {len(missing)} 卷: {missing}')
    else:
        print('全部完整')
```

4. 发现缺失立即补跑

## 三、已知问题及处理

### 3.1 超大循环超时丢失试卷

- **现象**：`for batch in ...` 循环中批量调用 create.py，超时后已完成的批次文件正常保存，但未执行的批次全部丢失
- **处理**：补跑缺失范围，并确认每个 `printf "n\n" | python` 调用单独执行，不要 & 连写大量批次

### 3.2 修复死循环

- **现象**：窄考点（极重要/计算类）生成后 QA 检测到大量题干重复，修复环节重出题目时 AI 持续返回空解析，导致 `_repair_qc_issues_targeted` 进入无效修复循环
- **处理**：由用户自行调整 `prompts.py` 修复逻辑

### 3.3 省份正则不匹配

- **现象**：`planning.py` 第93行正则 `([\u4e00-\u9fa5]+(?:省|市))` 无法匹配"内蒙古自治区"
- **修复**：增加 `|自治区` 匹配项

### 3.4 出版社解析

- **现象**：教材信息行写"重大版·第4版"，但 `_parse_edition` 不识别"重大版"格式，文件命名回退为"高教版"
- **待处理**：扩展出版社缩写映射

### 3.5 文件名特殊字符

- **现象**：主题名含 `/`（如"D/A转换"），Windows 将其解析为路径分隔符，导致保存失败
- **处理**：将规划表中的 `/` 改为 `-`

### 3.6 新题型支持

- **现象**：`prompts.py` 仅解析 单选/多选/判断/综合/填空，不识别"计算""简答"等自定义题型
- **处理**：由用户在 prompts.py 中增加对应题型解析和格式模板
