# 考纲百套卷

`考纲百套卷` 是一套用于批量生产对口升学 / 高职分类考试训练卷的项目。它把“规划表”作为核心输入，按卷号生成、质检、修复、导出 Word，并最终整理为可交付文件。

这份 README 的目标是：**让代码基础薄弱的人，也能看懂这个项目的工作流程，知道文件放哪里、运行哪个入口、生成结果去哪里看。**

---

## 一句话理解这个项目

你可以把本项目理解成一条试卷生产流水线：

```text
准备规划表 / 映射表 / 待质检试卷
        ↓
运行 python create.py
        ↓
选择模式 1 或模式 2
        ↓
系统拆题、质检、修复、生成 Word
        ↓
到 04_生成输出/生成结果/ 查看最终 DOCX 和 ZIP
```

## 项目适合谁使用

### 适合

- 需要批量生成考纲训练卷的人；
- 需要把人工组卷 DOCX 统一质检、修复、排版的人；
- 需要从规划表和学科网题库自动拉题的人；
- 需要追踪每卷“为什么失败 / 修了什么 / 输出到哪里”的人。

### 不适合

- 只想临时手写一份单卷、不需要批处理的人；
- 没有规划表，也没有待质检 DOCX，却希望系统凭空知道出卷范围的人；
- 希望完全不人工复核就直接交付所有卷的人。系统能修很多格式和结构问题，但仍可能留下需要人工确认的问题。

## 一、项目核心概念

### 1. 规划表

规划表是本项目最重要的输入文件。它决定：

- 哪个省份、哪个考类；
- 有哪些课程；
- 有哪些专题；
- 每个考点对应哪些卷号；
- 每卷需要哪些题型、多少题、多少分。

规划表通常放在：

```text
04_生成输出/生产规划/{省份} {考类}/
```

例如：

```text
04_生成输出/生产规划/重庆市 电子信息类/重庆市_电子信息类_考点规划总表.xlsx
```

### 2. 映射表

映射表负责把规划表里的中文考点，转换成学科网 API 能识别的知识点 ID。

它通常放在：

```text
04_生成输出/生产规划/{省份} {考类}/{省份}_{考类}_映射表.xlsx
```

例如：

```text
04_生成输出/生产规划/重庆市 电子信息类/重庆市_电子信息类_映射表.xlsx
```

如果某卷在映射表里标记为 `AI生成`，系统会跳过学科网 API 拉题，改为直接调用 AI 命题。

### 3. 细目表

专题训练卷和课程综合卷通常需要细目表。细目表说明每一道题考什么、题型是什么、难度是什么、对应哪个考点。

考点训练卷一般不需要单独细目表，系统会按规划表自动处理。

### 4. 组卷待质检

这里放等待处理的 DOCX 或 TXT。

```text
04_生成输出/组卷待质检/
```

按省份和考类分类时，常见路径是：

```text
04_生成输出/组卷待质检/重庆市 电子信息类/
```

模式 1 会优先从这里找 DOCX / TXT。

模式 2 拉题后，也会先把生成的 DOCX 放到这里，然后继续进入质检修复流程。

## 二、两种主模式的区别

## 模式 1：质检修复模式

### 适用场景

你已经有一份或多份 DOCX / TXT，需要系统帮你：

1. 拆成一道一道题；
2. 检查题目是否缺题干、缺答案、缺解析、格式错误等；
3. 对可自动修复的问题调用 AI 修复；
4. 生成解析版 DOCX；
5. 生成原卷版 DOCX；
6. 打包 ZIP；
7. 输出质检报告。

### 输入放哪里

把待处理文件放到：

```text
04_生成输出/组卷待质检/{省份} {考类}/
```

例如：

```text
04_生成输出/组卷待质检/重庆市 电子信息类/
```

文件名最好包含卷号，例如：

```text
第1卷 电路组成与电路模型.docx
第29卷 电路基本概念与定律.docx
```

### 实际流程

```text
选择模式 1
  ↓
读取规划表
  ↓
选择卷号
  ↓
从 组卷待质检 里找对应 DOCX / TXT
  ↓
拆题
  ↓
修复前质检
  ↓
AI 定向修复
  ↓
修复后质检
  ↓
生成解析版 DOCX
  ↓
生成原卷版 DOCX 和 ZIP
  ↓
写入生成结果、质检报告、运行记录
```

### 注意

当前代码中，模式 1 如果找不到对应 DOCX / TXT，会尝试先按规划表调用 API 拉题；如果 API 也没有生成可用文件，还可能询问是否用 AI 兜底生成整卷。

所以模式 1 的核心是“质检修复”，但它并不是完全只处理已有 DOCX。

---

## 模式 2：API 拉题模式

### 适用场景

你还没有人工组好的 DOCX，希望系统根据规划表和映射表，从学科网 API 拉题。

### 必备条件

模式 2 通常需要：

1. 规划表；
2. 映射表；
3. `02_配置资源/config.json` 中有可用配置；
4. 学科网 Cookie 可用；
5. 对应课程、题型、知识点能解析到 API 参数。

### 实际流程

```text
选择模式 2
  ↓
读取规划表
  ↓
读取映射表
  ↓
选择卷号
  ↓
按课程 / 题型 / 知识点调用学科网 API
  ↓
按题源、年份、重复度等规则筛题
  ↓
题量不足时调用 AI 补题
  ↓
生成待质检 DOCX
  ↓
自动进入模式 1 的质检修复流程
  ↓
生成最终 Word 和 ZIP
```

### 重要理解

模式 2 不是“只拉题”。它的完整链路是：

```text
API 拉题 → 生成待质检 DOCX → 质检 → 修复 → 生成正式文件
```

也就是说，模式 2 最后还是会走模式 1 的质检修复流程。

## 三、独立质检入口：check.py

如果你只想检查 `组卷待质检` 中的 DOCX / TXT，可以运行：

```bash
python check.py
```

它会让你选择考类和文件，然后执行：

```text
选择文件
  ↓
拆题
  ↓
结构化质检
  ↓
输出 Markdown 质检报告
```

它和 `create.py` 的区别：

| 入口 | 用途 |
|---|---|
| `python create.py` | 主生产入口，会质检、修复、生成最终文件 |
| `python check.py` | 独立质检入口，主要用于快速检查待质检文件 |

日常正式生产，优先使用：

```bash
python create.py
```

## 七、系统会检查什么

质检规则主要在：

```text
01_工具脚本/质检/rules.py
```

常见检查包括：

| 检查类型 | 示例 |
|---|---|
| 单题缺失 | 缺题干、缺答案、缺解析、缺选项 |
| 题型格式 | 题型不规范、答案格式错误、选项结构异常 |
| 内容质量 | 解析过短、网页残留、答案在题干中暴露 |
| 整卷结构 | 题号不连续、题量不符合规划、答案分布失衡 |
| 跨卷重复 | 课程综合卷之间重复率过高 |
| 配图风险 | 题干说“如图”但没有图片，高风险图题未配图 |

## 八、系统会怎么修复

修复模块主要在：

```text
01_工具脚本/生成器/regenerator.py
```

系统会对可自动修复的问题进行有限轮次修复，例如：

- 补充解析；
- 修正答案；
- 修正题型；
- 修正选项；
- 清理明显格式问题；
- 调整判断题答案格式；
- 修复答案分布异常。

注意：

> 修复不是无限循环，也不是整卷无条件重写。系统会尽量“定向修复”有问题的题目，避免把原本可用的内容改坏。

如果修复后仍有问题，系统会在报告中保留人工复核提示。

## 九、模式 2 的 API 拉题规则简述

模式 2 会按以下思路拉题：

1. 从规划表读取卷号、课程、题型、题量；
2. 从映射表读取知识点 ID；
3. 解析课程 ID 和题型 ID；
4. 按题型和难度请求学科网 API；
5. 对返回题目做过滤和去重；
6. 对同专题的考点训练卷使用“专题小题库”，减少重复；
7. API 题量不足时，调用 AI 补题；
8. 如果题量命中率低于 40%，可能进入半量策略，并提示每题分值 ×2；
9. 生成 DOCX 到 `组卷待质检`；
10. 自动继续质检修复。

API 拉题相关代码主要在：

```text
01_工具脚本/学科网API拉题移植版/
```

重点文件：

| 文件 | 作用 |
|---|---|
| `query_questions.py` | 组织并发送 API 请求 |
| `kpoint_resolver.py` | 解析课程、题型、知识点映射 |
| `api_pull_core.py` | 内容过滤、拉题辅助逻辑 |
| `html_content_converter.py` | 把 API 返回的 HTML 转成文本 |

## 十一、完整生产流程示例

以下以“重庆市 电子信息类”为例。

### 第 1 步：准备规划文件

确认目录中有：

```text
04_生成输出/生产规划/重庆市 电子信息类/重庆市_电子信息类_考点规划总表.xlsx
04_生成输出/生产规划/重庆市 电子信息类/重庆市_电子信息类_映射表.xlsx
```

专题训练卷和课程综合卷还应有对应细目表 DOCX。

### 第 2 步：启动系统

```bash
python create.py
```

### 第 3 步：选择模式

如果已有 DOCX：

```text
选择 1
```

如果需要 API 拉题：

```text
选择 2
```

### 第 4 步：选择规划表

如果系统发现多个规划表，会列出编号，让你选择。

### 第 5 步：选择卷号

可以输入：

```text
1
```

或：

```text
1-10
```

或：

```text
1,3,5
```

或：

```text
all
```

### 第 6 步：查看生成前检查

系统会显示：

- 选择了哪些卷；
- 哪些卷还没生成；
- 是否找到人工组卷 DOCX；
- 细目表是否齐全；
- 映射表是否覆盖；
- 配置 / API 是否可用。

如果有异常，系统会询问是否继续。

### 第 7 步：等待处理完成

系统会逐卷处理，输出过程信息。

### 第 8 步：检查结果

优先看：

```text
04_生成输出/生成结果/
04_生成输出/质检报告/
04_生成输出/运行记录/
```

## 十二、常见问题

### 1. 运行后提示找不到规划表怎么办？

检查是否存在：

```text
04_生成输出/生产规划/{省份} {考类}/xxx考点规划总表.xlsx
```

文件名中通常需要包含：

```text
考点规划总表
```

### 2. 模式 1 提示找不到待质检 DOCX 怎么办？

检查 DOCX 是否放在：

```text
04_生成输出/组卷待质检/{省份} {考类}/
```

并且文件名最好包含卷号，如：

```text
第1卷 xxx.docx
```

### 3. 模式 2 提示映射表缺失怎么办？

检查是否存在：

```text
04_生成输出/生产规划/{省份} {考类}/{省份}_{考类}_映射表.xlsx
```

例如：

```text
04_生成输出/生产规划/重庆市 电子信息类/重庆市_电子信息类_映射表.xlsx
```

### 4. API 拉题失败怎么办？

优先检查：

- `02_配置资源/config.json` 是否有 API key；
- 是否有可用学科网 Cookie；
- 映射表是否有知识点 ID；
- 课程名、题型名能否解析；
- 运行记录里写的失败原因。

### 5. 生成了 Word，但还有质检问题怎么办？

看对应的修复后质检报告：

```text
04_生成输出/质检报告/
```

如果报告中仍有失败或警告，需要人工复核。系统不会保证所有题都能自动修到完全无问题。

### 6. 为什么有些卷会进入 AI 生成？

常见原因：

- 映射表里该卷标记为 `AI生成`；
- 映射表没有知识点 ID；
- API 拉题不足；
- 没有找到人工 DOCX，API 也无法生成完整试卷。

### 7. 为什么模式 2 生成后还会质检？

因为 API 拉到的题也可能存在格式、答案、解析、配图或重复问题。项目的设计原则是：

```text
任何来源的题目，都必须进入质检修复链路。
```

## 十三、给代码基础薄弱使用者的建议

如果你只是使用项目，不打算改代码，请记住这 5 件事：

1. **主入口只有一个优先选择：** `python create.py`
2. **已有 DOCX 选模式 1，没有 DOCX 想拉题选模式 2。**
3. **输入主要放两个地方：**
   - 规划类文件放 `04_生成输出/生产规划/`
   - 待质检 DOCX 放 `04_生成输出/组卷待质检/`
4. **输出主要看三个地方：**
   - 最终文件：`04_生成输出/生成结果/`
   - 质检报告：`04_生成输出/质检报告/`
   - 运行记录：`04_生成输出/运行记录/`
5. **失败时不要只看终端，优先看运行记录和质检报告。**
