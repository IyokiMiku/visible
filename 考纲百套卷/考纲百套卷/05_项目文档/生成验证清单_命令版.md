
### 2.2 运行生成流程

```bash

```


- [ ] 第一套卷完成后，题目是否被标记为已用

- [ ] 同 `year + source / 试卷名` 组是否最多 3 道

```bash

```

## 7. 40% 分界验证

选择一套题量明显不足的卷：

- [ ] 当初始可用题量 `< 40%` 时，是否只补到规划的一半
- [ ] 是否出现“每题分值 ×2”的提示
- [ ] 当初始可用题量 `>= 40%` 时，是否补齐到规划数
- [ ] 是否不再提示分值翻倍

## 8. 过量题目验证

选择一套题目明显超出的卷：

- [ ] 是否裁剪到规划题量
- [ ] 裁剪后是否仍可通过质检
