# 配图风险前置检测系统 — 实现总结

## What was done

实现了5环节配图风险前置系统，将配图风险从"审核阶段被动发现"提前到"规划阶段主动识别"。核心思路：在关键词匹配的基础上，按课程→专题→考点三级递进判定风险等级，贯穿规划→出题→质检→QC全流程。

## Key changes

### 新增文件
| 文件 | 说明 |
|---|---|
| `01_工具脚本/生成器/image_risk.py` | 核心检测模块（~300行），含关键词库、风险判定、缺图检测、报告生成 |

### 修改文件
| 文件 | 变更 |
|---|---|
| `01_工具脚本/生成器/runner.py` | PreflightResult 新增 `image_risk_warnings`；preflight 阶段自动扫描+展示；善后生成配图风险报告 |
| `01_工具脚本/质检/rules.py` | `run_quality_checks()` 集成逐题缺图检测，高/中风险卷自动触发 |

## 5环节覆盖

| # | 环节 | 实现 | 触发时机 |
|---|------|------|----------|
| 1 | 规划表扫描 | `classify_paper_risk()` — 三级递进判定（课程→专题→考点关键词） | preflight |
| 2 | Config标记 | `PaperPlan.image_risk_level/score/reasons` — 运行时附加属性 | preflight |
| 3 | 出题规避 | preflight 对话展示 ⚠️ 高风险卷警告 | 生成前 |
| 4 | auto_review | `check_image_requirement()` — 题干"如图"无图→failed，高风卷无图→warning | 逐题质检 |
| 5 | QC优先 | `配图风险检测报告.md` — 按风险等级排序的检查清单 | 整批完成 |

## 实测数据（重庆市电子信息类 94卷）

```
高风险  5卷  (70分) — 数字电路/逻辑电路等含"电路图"考点
中风险 39卷  (50分) — 含"结构/工作原理/组成"等关键词
无风险 50卷  (0分)  — 理论概念类考点
```

## 风险等级判定规则

- **high (≥70)**：课程名命中机械制图等，或专题名命中装配图/电路图等，或考点内容含"读图/如图所示"等
- **medium (≥40)**：考点含"结构/工作原理/故障诊断"等中风险关键词
- **none**：纯理论知识点，不涉及图形
