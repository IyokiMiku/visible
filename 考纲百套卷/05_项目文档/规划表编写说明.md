# 考纲百套卷规划表编写说明

> 本文档用于指导 AI 根据考纲 PDF 一次性产出三份产物：**考点规划总表（xlsx）**、**细目表（docx）**、**映射表（xlsx）**。阅读本文档后，无记忆 AI 应能独立完成全部三份文件的生成。

---

## 一、你需要产出的三份文件

生成考纲百套卷的第一步是从考纲 PDF 出发，在 `04_生成输出/生产规划/{省份} {考类}/` 目录下产出以下文件：

| 序号 | 文件名 | 格式 | 作用 |
|:--:|------|:--:|------|
| 1 | `{省份}_{考类}_考点规划总表.xlsx` | xlsx | 定义全部考点、卷号、题型结构 |
| 2 | `专题训练卷_第x卷_{省份}_{专题名}.docx` | docx | N 份，每份对应一个专题训练卷的逐题约束 |
| 3 | `课程综合卷_第x卷_{省份}_{课程名}.docx` | docx | 每门课程 2-3 份，合起来覆盖该课程全部考点 |
| 4 | `{省份}_{考类}_映射表.xlsx` | xlsx | 考点文本 → 学科网知识点 ID 对照表 |

## 二、开始前的准备

### 2.1 必须收集的信息

在动手之前，先向用户确认以下信息：

1. **省份**（如"重庆市"、"四川省"）
2. **考类名称**（如"电子信息类"、"汽车类"）
3. **考试科目表名称**——用户必须提供，不可自行编造。示例：
   ```
   重庆市2025年中等职业学校毕业生进入普通高校学习专业基础课和专业课考试科目表
   ```
4. **知识树 CSV 文件**——从 `02_配置资源/学科网映射/knowledge_points/` 目录读取对应课程的知识树。若目录为空，跳过映射表生成，但仍需输出规划表和细目表。

### 2.2 输入文件

- **考纲 PDF**：`03_项目数据/考试说明/{年份}{省份}{考类}考试说明.pdf`
- **模板参考**：`河南省_机电与制造类_考点规划总表.xlsx`（用于仿照表格结构）
- **知识树 CSV**：`02_配置资源/学科网映射/knowledge_points/{课程名} - 知识树.csv`
- **知识点题目数量文档**：`05_项目文档/知识点题目数量/{课程名}_知识点题目数量.md`（用于窄考点判断）

### 2.3 知识点题目数量文档的准备与使用

在规划表编写之前，需确保考纲涉及的所有课程都有对应的知识点题目数量文档。

**生成文档：**

1. 检查 `05_项目文档/知识点题目数量/` 下是否已有对应课程的数量文档
2. 缺失的课程，运行以下命令生成（逐个课程，避免并发过快触发 API 限流）：
   ```bash
   python 01_工具脚本/学科网API拉题移植版/kpoint_type_summary.py "课程名1" "课程名2" ...
   ```
3. 已有数量文档的课程无需重新生成（脚本自动跳过）

**阅读文档：**

数量文档是 Markdown 格式的知识树，每份文档结构如下：

```
# 电工技术基础与技能 知识点题目数量汇总（课程名）

## 安全用电 (87636)                    ← L2 父节点，括号内是 kpointId
- 合计: 182 题                         ← 该节点所有叶子题型数量总和

| 节点ID | 知识点 | | 单选题 | 判断题 | ... | 合计 |   ← 叶子题型表格
|--------|--------||---:|---:|-----|-----:|
| 87637 | 人体触电常识 | 48 | 13 | ... | 79 |       ← L3 叶子，一行一个知识点
| 87638 | 触电的常见原因 | 5 | 2 | ... | 21 |

## 电路的基本概念 (87644)             ← 下一个 L2 父节点
- 合计: 529 题
...
```

**关键字段含义：**

| 字段 | 含义 | 用途 |
|------|------|------|
| `节点ID` | 知识树 kpointId，映射表 B 列填入的值 | 查 API 题目用 |
| `知识点` | 知识树节点名称 | 与 D 列文本匹配 |
| `单选题`~`作图题` | 该知识点下各题型可用题目总数 | 判断是否"窄" |
| `合计` | 所有题型题目数之和 | 窄考点首要判据 |

**如何根据数量文档判断窄考点：**

1. 在文档中找到与规划表 D 列文本对应的叶子知识点行（按名称模糊匹配）
2. 看该行的 `合计` 列——合计 < 80 → 窄考点
3. 合并后重新查文档中合并后的叶子合计，确保 ≥ 80（< 80 则走减半通道）

**窄考点合并规则（5条硬约束，缺一不可）：**

| 序号 | 约束 | 说明 |
|:--:|------|------|
| 1 | **内容相近** | 合并的前提不是"题少就合"，而是"内容相近才合"。两个知识点只要内容方向不同，无论题量多少都不可合并。例：同一个 L2 下有"欧姆定律"和"电阻定律"，虽题都很少，但一个讲电压电流关系、一个讲材料电阻特性，内容不相近 → 不合并 |
| 2 | 同一 L2 父节点 | 只能在同一个知识树 L2 父节点下的叶子之间合并，不跨 L2（因为不同 L2 代表完全不同的知识方向） |
| 3 | 考纲中相邻 | 只合并考纲编排中相邻出现的知识点，不跳着合并 |
| 4 | 最多合并 3 个 | 一次合并的窄考点不超过 3 个 |
| 5 | 合并后 ≥ 80 | 合并后所有叶子合计 ≥ 80 → 正常出卷；< 80 → 走减半通道（题量减半、每题分值×2） |

**合并后效果：**
- 多个窄考点合并为一个新考点，出 1 卷（不是多个窄考点各出一卷共享题库）
- 新考点名称使用概括性统称，C 列知识点总数 4-10 个
- 无法合并的独立窄考点（不符合上述任一约束）→ 直接走减半通道

**判定流程：**

```
读数量文档 → 找合计 < 80 的叶子节点
  ├── 是否同 L2 父节点下、考纲相邻、内容相近？
  │     ├── 是 → 合并（≤3个）
  │     │     ├── 合并后合计 ≥ 80 → 正常出 1 卷
  │     │     └── 合并后合计 < 80 → 减半通道
  │     └── 否 → 独立窄考点 → 减半通道
  └── 无法与任何相邻考点合并 → 减半通道
```

**示例：**

> 例1：D列="了解电路的基本组成及功能，理解电路模型" → 找到 `## 电路的组成与电路模型 (87645)`，合计 529
> → 529 ≥ 80 → **不是窄考点**，可独立成卷
>
> 例2：同一 L2"电路基本物理量"下，电流概念 32 题 + 电压概念 28 题 + 电位概念 35 题，考纲相邻，内容相近
> → 32+28+35=95 ≥ 80，符合全部 5 条约束 → **合并为 1 卷**
>
> 例3：同一 L2"直流电路分析"下，欧姆定律 25 题 + 电阻定律 22 题，考纲相邻但内容不相近
> → 违反约束 1 → **不合并**，各自走减半通道

### 2.4 输出路径

```
04_生成输出/生产规划/{省份} {考类}/
├── {省份}_{考类}_考点规划总表.xlsx
├── {省份}_{考类}_映射表.xlsx
├── 专题训练卷_第29卷_{省份}_{专题名}.docx
├── 专题训练卷_第30卷_{省份}_{专题名}.docx
├── ...
├── 课程综合卷_第31卷_{省份}_{课程名}.docx
└── ...
```

## 三、第一步：产出考点规划总表（xlsx）

### 3.1 表格结构

| 行号 | 内容 |
|:--:|------|
| 1 | 标题行（合并 A-J）：`《{省份}{考类}》系列卷考点规划总表（百套卷）` |
| 2 | 考试科目表名称（A 列填写，不合并）：用户提供的官方考试科目表全称 |
| 3 | 考试信息（合并 A-C 写"每卷考试信息"，D-E 写"每卷分值 100分"，F-G 写"考试时间 60分钟"） |
| 5 | 副标题（合并 A-J）：`《{省份}{考类}》系列卷考点规划总表` |
| 6 | 表头行（10 列，见 3.2） |
| 7+ | 数据行 |
| 末+1 | 合计行 |
| 末+3 | 题型汇总表 |

### 3.2 十列表头

| 列 | 表头 | 含义 | 合并 |
|:--:|------|------|:--:|
| A | 知识模块 | 课程/书名 | ✅ 同课程 |
| B | 专题名称 | 考纲一级标题 | ✅ 同专题 |
| C | 考点名称 | 考点训练卷名称 | ❌ |
| D | 掌握/理解/了解考点内容 | 考纲细碎知识点 | ❌ |
| E | 考点训练卷数量 | 固定 1 | ❌ |
| F | 考点训练卷卷号 | "第x卷" | ❌ |
| G | 专题训练卷数量 | 固定 1 | ✅ 同专题 |
| H | 专题训练卷卷号 | "第x卷" | ✅ 同专题 |
| I | 课程综合卷数量 | 2 或 3 | ✅ 同课程 |
| J | 课程综合卷卷号 | "第x-y卷" | ✅ 同课程 |

### 3.3 数据行编写规则

**层级关系**：A ⊃ B ⊃ C ⊃ D（课程包含专题包含考点包含知识点）

**D 列约束**：
- 来源：仅考纲 PDF，不添加考纲外知识点
- 每条知识点以"1. 了解/理解/掌握 xxx"格式开头
- **末尾不得加句号或任何标点**

**命名规则**：
- A 列：考纲中的课程名/书名
- B 列：考纲一级标题/单元名，≤15 字
- C 列：AI 归纳的考点名称，≤15 字，可直接作卷名
- C 列覆盖 2-5 个 D 列知识点

**窄考点判定与合并规则**（5 条硬约束，全部满足才合并）：

窄考点判定：阅读知识点数量文档，对应叶子知识点的**合计 < 80** → 窄考点。

合并须同时满足以下 5 条约束：

| 序号 | 约束 | 说明 |
|:--:|------|------|
| ① | **内容相近**（最高优先级） | 前提不是"题少就合"，而是"内容相近才合"。两个知识点只要内容方向不同，无论题量多少都不可合并。例："欧姆定律"（电压电流关系）和"电阻定律"（材料电阻特性）虽属同一 L2 且题都很少，但内容不相近 → 不合并 |
| ② | 同一 L2 父节点 | 只能在知识树同一 L2 父节点下的叶子之间合并，不跨 L2 |
| ③ | 考纲中相邻 | 只合并考纲编排中相邻出现的知识点，不跳着合并 |
| ④ | 最多合并 3 个 | 一次合并的窄考点不超过 3 个 |
| ⑤ | 合并后 ≥ 80 | 合并后所有叶子合计 ≥ 80 → **正常出卷**；< 80 → **减半通道**（题量减半、每题分值×2） |

**合并后效果：**
- 多个窄考点合并为 **1 个新考点，出 1 卷**（不是多卷共享题库）
- 新考点名称使用概括性统称，C 列知识点总数 4-10 个为宜
- 无法合并的独立窄考点（不符合任一约束）→ 直接走减半通道
- 合并导致总卷数减少时，按序号重新编排卷号，确保全局连续无跳号

**判定流程：**

```
读数量文档 → 找合计 < 80 的叶子节点
  ├── 同 L2、考纲相邻、内容相近？
  │     ├── 是 → 合并（≤3个）
  │     │     ├── 合并后合计 ≥ 80 → 正常出 1 卷
  │     │     └── 合并后合计 < 80 → 减半通道
  │     └── 否 → 独立窄考点 → 减半通道
  └── 无相邻考点可合并 → 减半通道
```

**示例：**

> 例1（正常合并）：同一 L2"电路基本物理量"下，电流概念 32 题 + 电压概念 28 题 + 电位概念 35 题
> → 考纲相邻、内容相近（均属基本物理量）、32+28+35=95 ≥ 80、≤3 个
> → ✅ 满足全部 5 条约束 → **合并为 1 卷"电路基本物理量"**
>
> 例2（拒绝合并）：同一 L2 下，欧姆定律 25 题 + 电阻定律 22 题
> → 考纲相邻但内容不相近（一个讲 U=IR、一个讲 R=ρL/S）
> → ❌ 违反约束① → **不合并**，各自走减半通道
>
> 例3（合并但不够）：同一 L2 下，2 个内容相近知识点合计 25+18=43
> → 考纲相邻、内容相近、≤3 个，但 43 < 80
> → ❌ 违反约束⑤ → **合并为 1 卷，走减半通道**

**合并禁区**（新增）：
- ❌ 禁止跨专题合并：只能在同一个专题（B 列）内部、相邻考点之间合并，不得跨越专题边界
- ❌ 禁止跨课程合并：不同课程（A 列）的考点绝不可合并
- ❌ 禁止为凑卷数强行拆分内容充实的考点：若考点内容独立性强、各维度出题角度充足，必须保留独立考点以覆盖完整出题范围
- 合并必须保持课程内容的内在逻辑和编排顺序，不得破坏知识递进关系

**kpoint 合并导致的串知识点防范**（API 拉题阶段）：
- 当窄考点合并后，映射表中会使用合并后的 kpointId 集合拉取 API 题目。此时可能出现串知识点问题——例如考点为"冷却系统"，但 API 返回了"润滑系统"的题目（因为两者共用合并后的 kpointId 集合）
- 防范机制：在 API 拉题后、入选前，调用 `_content_filter` 基于考点名称与题干关键词做互斥过滤
- 关键词映射表按课程维护，定义互斥知识点对（如冷却↔润滑、配气↔曲柄连杆等）
- 过滤后题量不足时，由 AI 补题而不做宽泛兜底拉取
- 详见 `01_工具脚本/学科网API拉题移植版/api_pull_core.py` 中的 `apply_content_filter()`

**专题考点数约束**（新增）：
- 每个专题（B 列）涵盖 1~4 个考点（C 列行数）
- 多于 4 个考点时需要合并：将 C 列中知识点相近或同属一个知识块的考点合并为一个，合并后 C 列名称使用概括性统称
- 独立考点拆分原则：如果某个专题下考点之间内容独立性强、每个考点知识点均充足（D 列 ≥4 个知识点），必须保留独立考点以覆盖完整出题范围，不能为了避免超限而压缩考点粒度。拆分后总数仍受"每专题 1~4 个考点"上限约束，若拆分后超过 4 个，优先保留权重高（掌握类多）的考点独立，其余合并
- 此约束与窄考点判断规则为双向互补：
  - 窄考点判断（自下而上）：知识点不足以支撑独立成卷 → 向上合并（考点数减少）
  - 专题约束（自上而下）：一个专题下考点过多 → 合并相近考点（考点数收敛至 ≤4）

**卷号编排**（全局连续，跨课程不重置）：
1. 先排所有考点训练卷（F 列，逐行 +1）
2. 再排所有专题训练卷（H 列，按专题 +1）
3. 最后排课程综合卷（J 列，每课程 2-3 卷连续）

**合并规则**：
- A 列同课程合并、B 列同专题合并、G/H 同专题合并、I/J 同课程合并
- C/D/E/F 不合并

### 3.4 题型汇总表

合计行下方，为每门课程写一个题型表：

```
各课程题型结构（100分/60分钟，按真题等比缩放）

{课程名}（{原始分}→100，{缩放方式}）  题型      题数  每题分值  小计分值  总分
{课程名}                              单选题    x题   x分/题   xx分     100分
                                      判断题    x题   x分/题   xx分
                                      ...
```

每课程总分必须 100 分。若原真题总分非 100，写缩放关系如"（130→100，×0.769）"。

## 四、第二步：产出细目表（docx）

### 4.1 哪些卷需要细目表

仅**专题训练卷**和**课程综合卷**需要细目表。考点训练卷不需要。

### 4.2 细目表内容

每个细目表是一个 docx 文件，内部是一个表格，每行对应一道题：

| 字段 | 含义 | 说明 |
|------|------|------|
| 卷别 | 专题训练卷 或 课程综合卷 | |
| 卷号 | 如"第29卷" | |
| 地区 | 如"重庆市" | |
| 课程名称 | 规划表 A 列值 | |
| 专题名称 | 规划表 B 列值 | 课程综合卷可写"课程综合" |
| 题号 | 本卷第几题 | 连续，不跳号 |
| 题型 | 单选题/多选题/判断题/填空题/简答题/计算题/综合题 | 符合题型汇总表结构 |
| 难度 | 简单/适中/困难 | |
| 考查内容 | 一句话说明本题考什么 | 不写完整题干 |
| 对应考点 | 规划表 C 列考点名称 | 必须可追溯 |
| 对应知识点 | 规划表 D 列知识点 | 可多个 |
| 考纲要求 | 掌握/理解/了解 | |
| 出题意图 | 为什么这样安排 | |

### 4.3 题量分配原则

| 知识点类型 | 权重 | 出题建议 |
|------|:--:|------|
| 掌握类、计算类、核心公式/定律 | 高 | 2+ 题，可设适中/困难 |
| 理解类、常见概念/原理 | 中 | ≥1 题 |
| 了解类、识记类、低频 | 低 | 1 题或合并考查 |

### 4.4 覆盖要求

- **专题训练卷**：单卷必须覆盖该专题下全部考点
- **课程综合卷**：2-3 套合起来覆盖该课程全部考点，单套可只覆盖部分

### 4.5 难度分布

建议：简单 80% : 适中 10% : 困难 10%

### 4.6 命名规则

```
专题训练卷_第{x}卷_{省份}_{专题名}.docx
课程综合卷_第{x}卷_{省份}_{课程名}.docx
```

### 4.7 注意事项

- **D 列编号与最终 DOCX 无关**：D 列的"1. 2. 3."序号仅用于规划表内部索引，生成试卷的编写说明会自动从 1 重新编号并用分号连接，无需手动调整 D 列编号
- **细目表不涉及 kpoint 匹配**：细目表中"对应考点"和"对应知识点"填写规划表 C/D 列原文即可，不需要填入数字 ID——那个是映射表的职责

## 五、第三步：产出映射表（xlsx）

### 5.1 映射表是什么

将每套试卷与学科网知识树中的数字 ID 建立对应关系。后续 API 拉题时通过这个表把试卷对应的中文考点转成数字 ID 查询，实现"按卷拉题"。

**核心原则：只有考点训练卷需要 AI 逐行匹配。专题训练卷和课程综合卷的映射由代码自动聚合，无需 AI 单独处理。考纲考点与知识树节点无法匹配时，标记「AI生成」而非强行指向不相关的节点。**

### 5.2 前置条件

必须读取对应课程的知识树 CSV：`02_配置资源/学科网映射/knowledge_points/{课程名} - 知识树.csv`

CSV 格式：
```
id,parent_id,name,level
50401,0,电工技术基础与技能,1
5040101,50401,电路基本概念与定律,2
504010101,5040101,电路的基本组成,3
...
```

### 5.3 映射表结构

| 列 | 表头 | 内容 |
|:--:|------|------|
| A | 试卷序号 | 试卷卷号，如"第1卷""第29卷""第31-33卷" |
| B | 知识点 ID | 学科网 kpointId，多个用半角逗号分隔，如 `504010101,504010102` |
| C | 映射方式 | `AI匹配`（考点训练卷，知识树匹配成功）/ `AI生成`（考点训练卷，知识树无法匹配，由 AI 直接命题）/ `聚合`（专题训练卷 / 课程综合卷，代码自动聚合） |
| D | 备注 | 聚合卷写明来源，如"聚合自第1.2.3卷" / "聚合自第1-28卷" |

**A 列必须覆盖全部试卷**——不仅包含考点训练卷（规划表 F 列有记录的行），还包含专题训练卷（H 列）和课程综合卷（J 列）。

### 5.4 映射方法——分两步走

#### 第一步（AI 完成）：匹配考点训练卷

逐行读取规划表 D 列内容 → 对每条考点在知识树 CSV 中查找匹配 → 填入 B 列。**这是唯一需要 AI 参与的步骤。**

具体匹配过程：
1. 从 D 列文本中提取核心关键词（忽略"了解/理解/掌握/1. "等层级前缀和序号）
2. 在知识树中搜索包含该关键词的节点，按以下**层级优先级**选择：
   - **优先 L3 叶子节点**：先在 L3 叶子中精确匹配。匹配到的 L3 节点可能是一组，全部填入
   - **退到 L2 父节点**：仅当 D 列内容涵盖该 L2 节点下大量叶子（≥60%）且各叶子题目总数均不足时，才匹配到 L2 父节点
   - **禁止 L1 根节点**：不得匹配到 L1 根节点（课程级），因为范围过大，会拉取无关子方向的题目
3. 匹配成功 → 填入对应的数字 ID。一个考点可能对应多个知识树叶子节点，全部填入并用逗号分隔
4. 匹配失败 → B 列留空，C 列填 `AI生成`

**原则**：宁缺毋滥。不确定的宁可留空，不瞎填。

##### 匹配禁区（必须遵守）

以下情况**必须标记为 `AI生成`**，C 列填 `AI生成`，D 列备注写明原因，B 列留空：

| 情形 | 示例 | 处理 |
|------|------|------|
| 知识树无对应节点 | D列="掌握 C51 数据类型与运算"，知识树仅有汇编指令集 | `AI生成` |
| 知识树节点名称相似但内容不同 | D列="了解中断系统的 C51 编程"，知识树节点="中断系统（汇编）" | `AI生成` |
| 考点范围大于知识树覆盖 | D列="掌握单片机的综合应用设计"，知识树仅有基础IO/定时器节点 | `AI生成` |
| 匹配到 L1 根节点 | D列="电路组成与电路模型"，匹配到 L1 节点"电路的基本概念" | 应退到 L2 或 L3 叶子；若均无法匹配则 `AI生成` |

**绝对禁止的行为**：
- ❌ 知识树无 C51 → 强行填入汇编指令集节点 ID
- ❌ 知识树节点名称相似但内容不同 → 勉强匹配
- ❌ 考点是"综合应用"但知识树只有基础节点 → 填入基础节点 ID
- ❌ 任何"差不多就行"的模糊匹配——只能是精确内容匹配
- ❌ 匹配到 L1 根节点（课程级）——只能匹配 L3 叶子或 L2 父节点

> ⚠️ **常见错误**：看到 D 列"电路组成与电路模型"就匹配到 L1"电路的基本概念"(87644)——这是典型错误。正确做法：优先匹配 L3 叶子（87646+87647+87648），或退到 L2"电路的组成与电路模型"(87645)。L1 节点包含不相干的电流电压/电功率等方向，会导致 API 拉题严重串知识点。

#### 第二步（代码完成）：聚合专题训练卷和课程综合卷

考点训练卷全部匹配完成后，代码自动按以下规则聚合：

```python
# 伪代码 — 展示聚合逻辑，不要求 AI 执行

# 专题训练卷聚合
for each B列专题:
    该专题下所有考点训练卷的B列ID → 合并去重 → 赋给对应的专题训练卷

# 课程综合卷聚合
for each A列课程:
    该课程下所有考点训练卷的B列ID → 合并去重 → 赋给该课程的全部综合卷
```

**聚合示例（重庆市电子信息类——电工技术基础与技能）：**

```
规划表（AI匹配，优先级L3叶子→L2父节点，禁止L1根节点）：
  第1卷（考点训练卷）→ C列：电路组成与电路模型 → B列：87646,87647,87648   ← L3叶子
  第2卷（考点训练卷）→ C列：电流电压与电位     → B列：87652,87653,87654,87655 ← L3叶子
  ...

聚合结果（代码自动生成）：
  第29卷（专题训练卷）→ 专题：电路基本概念与定律  → B列：第1.2...卷的ID合并去重    ← 聚合
  第30卷（专题训练卷）→ 专题：直流电路分析        → B列：第4.5...卷的ID合并去重    ← 聚合
  ...
  第31卷（课程综合卷）→ 课程：电工技术基础与技能  → B列：第1-28卷全部ID合并去重    ← 聚合
```

**注意**：同一课程的多卷课程综合卷（如第31-33卷）共享完全相同的知识点 ID 列表——它们覆盖的是同一门课的全部考点，只是题目不同。

### 5.5 命名与路径

```
04_生成输出/生产规划/{省份} {考类}/{省份}_{考类}_映射表.xlsx
```

例如：
```
04_生成输出/生产规划/重庆市 电子信息类/重庆市_电子信息类_映射表.xlsx
```

### 5.6 格式要求

- Sheet 名："知识点映射"
- 第 1 行：表头（试卷序号、知识点 ID、映射方式、备注）
- 第 2 行起：数据
- C 列填 `AI匹配` 或 `聚合`，使代码能区分两类记录
- 文件末尾如有未匹配的考点，另起一行写 `# 未匹配列表`，下列出未能匹配的试卷序号和 D 列原文

### 5.7 AI 生成映射表时的操作步骤

1. **先阅读知识点题目数量文档**：打开 `05_项目文档/知识点题目数量/{课程名}_知识点题目数量.md`，了解每个知识点的各题型可用数量。缺失则运行 `kpoint_type_summary.py` 生成
2. 打开规划表 xlsx，定位到数据行
3. 打开对应课程的知识树 CSV（`02_配置资源/学科网映射/knowledge_points/{课程名} - 知识树.csv`）
4. **只处理有 D 列内容的数据行**（即考点训练卷），逐行：
   - 提取 D 列核心关键词（去掉"了解/理解/掌握/1. 2. "等前缀）
   - 在知识树中搜索匹配，严格遵循 **L3 叶子 → L2 父节点** 的优先级，**禁止 L1 根节点**
   - 匹配成功 → B 列填 ID（多个叶子用逗号分隔），C 列填 `AI匹配`
   - 匹配失败 → B 列留空，C 列填 `AI生成`，D 列备注写原因
5. **规划表中专题训练卷和课程综合卷对应的行**（无 D 列内容）：
   - B 列留空（由代码聚合填充）
   - C 列填 `聚合`
   - D 列备注可留空（代码填充时会自动写入聚合来源）
6. 保存 xlsx，确保行数与规划表数据行数一致

### 5.8 标记「AI生成」后的运行时行为

当映射表 C 列标记为 `AI生成` 时，试卷生成系统会自动：

1. **跳过 API 拉题**——不再尝试从学科网题库获取题目
2. **直接进入 AI 命题流程**——调用大模型按编写规范独立出题
3. **后续质检/修复/Docx 输出流程不变**

因此，AI 生成映射表时只需确保 C 列正确标记，无需担心这些试卷的后续处理。代码会自动走 AI 命题通道。

## 六、生成顺序总结

```
第〇步：检查知识点题目数量文档
  05_项目文档/知识点题目数量/{课程名}_知识点题目数量.md
  缺失则运行 kpoint_type_summary.py 生成，已有则跳过
第一步：阅读考纲 PDF + 知识点题目数量文档，提取并评估全部知识点
第二步：根据窄考点判断规则，合并窄考点，生成考点规划总表 xlsx（规划表）
第三步：基于规划表，为每个专题训练卷生成细目表 docx
第四步：基于规划表，为每门课程生成 2-3 份课程综合卷细目表 docx
第五步：基于规划表 + 知识树 CSV，生成映射表 xlsx（AI 只匹配考点训练卷，专题/综合卷 B 列留空）
第六步（代码自动）：load_mapping_table() 加载时自动聚合专题/综合卷的 ID
```

全部文件输出到 `04_生成输出/生产规划/{省份} {考类}/`。

**注意**：第五步和第六步的分工——AI 只负责"考点文本 → 数字 ID"的语义匹配（考点训练卷）。匹配不上时标记 `AI生成` 而非强行填错 ID。专题训练卷和课程综合卷的聚合由代码在加载映射表时自动完成。AI 不需要计算专题/综合卷的 ID。

## 八、唯一事实源与配置漂移防控

### 8.1 问题背景

项目运行过程中可能同时存在多份"看起来都能描述试卷"的文件：

- 原始 xlsx 规划表
- `planning_table.json`
- `planning_cache.json`
- `shared/configs/{年份}/{省份}/{专业}/` 下的省份/专业配置
- 单卷 `volXX_config.json`

如果没有明确"谁说了算"，后续人工修改时容易在不同文件间产生不一致，最终出现**配置漂移**——无人知道哪个文件的数值是正确的。

### 8.2 事实源优先级规则

**规划表 xlsx 为最高优先级事实源。** 出卷范围、卷号、考点归属如有争议，以原始 xlsx 规划表为准。

各文件的事实源角色定义如下：

| 文件 | 角色 | 规则 |
|------|:--:|------|
| **xlsx 规划表** | 🔴 最高事实源 | 人工可读/可编辑。出卷范围、卷号、考点归属的唯一权威。任何 JSON/配置与 xlsx 冲突时，以 xlsx 为准。修改后必须重新转换并校验。 |
| `planning_table.json` | 🟡 运行时事实源 | 由 xlsx 转换生成。代码运行时以此文件为准读取出卷范围、卷号、考点。**每次 xlsx 修改后必须重新生成此文件**，禁止手工编辑。 |
| `shared/configs/{年份}/{省份}/{专业}/` | 🟡 运行时事实源 | 省份级规则（题型结构、满分、考试时间、卷头文案）的运行时事实源。**不与卷号/考点范围重叠**，与 `planning_table.json` 职责正交。 |
| `planning_cache.json` | ⚪ 派生缓存 | 仅供性能加速。可随时从 `planning_table.json` 重建。不应长期手工维护，不应作为事实源。 |
| `volXX_config.json` | ⚪ 单卷实例 | 单卷出题时的实例快照，从 `planning_table.json` + `shared/configs` 派生生成。**不应重复写入全局规则**（如题型结构、考点范围），仅记录该卷的运行时状态（如已出题数、质检状态）。 |

### 8.3 职责边界（正交切分）

```
┌─────────────────────┬──────────────────────────────┬─────────────────────────────────┐
│       职责          │          事实源              │             说明                │
├─────────────────────┼──────────────────────────────┼─────────────────────────────────┤
│ 出卷范围、卷号、考点│ xlsx 规划表 → planning_table │ 规划表独有。哪个考点归哪个卷、  │
│                     │    .json（运行时）           │ 卷号怎么排，全在这里决定。      │
├─────────────────────┼──────────────────────────────┼─────────────────────────────────┤
│ 题型结构、满分、时间│ shared/configs              │ 省份级，与考点无关。            │
│ 卷头文案、考试信息  │                              │ 同一省份/专业共享。            │
├─────────────────────┼──────────────────────────────┼─────────────────────────────────┤
│ 细目表（逐题约束）  │ docx 细目表文件             │ 规划表生成时一并产出。          │
│                     │                              │ 题型 + 难度 + 考点绑定。        │
├─────────────────────┼──────────────────────────────┼─────────────────────────────────┤
│ 知识点→学科网ID映射 │ 映射表 xlsx                 │ 考点训练卷AI匹配，其余聚合。    │
│                     │                              │ 治理规则见第五章。              │
├─────────────────────┼──────────────────────────────┼─────────────────────────────────┤
│ 出题质量/命题规则   │ 02_配置资源/命题要求/       │ 不随省份变化的核心命题规范。    │
│                     │ 02_配置资源/编写规范/        │                                 │
└─────────────────────┴──────────────────────────────┴─────────────────────────────────┘
```

### 8.4 修改工作流

**修改考点/卷号（涉及 xlsx）**：
```
1. 修改原始 xlsx 规划表（唯一可编辑源）
2. 运行 planning.py 重新转换 → planning_table.json
3. 删除 planning_cache.json（下次运行自动重建）
4. 校验：planning.py 打印统计，确认卷数/考点数与预期一致
```

**修改省份规则（不涉及 xlsx）**：
```
1. 修改 shared/configs/{年份}/{省份}/{专业}/ 下的配置文件
2. 无需动 planning_table.json 或 planning_cache.json
3. 校验：用该配置跑一卷，确认题型/满分/卷头正确
```

**禁止的操作**：
- ❌ 手工编辑 `planning_table.json` —— 修改会在下次 xlsx 转换时丢失
- ❌ 手工编辑 `planning_cache.json` —— 随时可能被重建
- ❌ 在 `volXX_config.json` 中硬编码题型结构或考点范围 —— 应读自 shared/configs
- ❌ xlsx 和 planning_table.json 同时修改而不重新转换 —— 必然产生漂移

## 九、流程唯一性与目录治理

### 9.1 问题背景

项目演进过程中可能出现新旧两套目录/流程并存的情况，例如：

- 旧版输出路径 `04_生成输出/生产规划/`
- 其它旧版/实验输出路径

如果新旧目录同时存在而没有明确"哪个是当前主流程"，操作者可能：
- 混用旧目录出卷、新目录出卷，输出散落各处
- 误用旧脚本/旧模板，导致格式不一致
- 旧版细目表与新流程不同步，不确定是否仍需交付

### 9.2 当前主流程声明

**本项目只有一套流程。** 输出统一根目录为 `04_生成输出/`，子目录按用途划分：

### 9.3 若出现旧版/实验目录

规则如下：

- **该目录不是当前主流程。** 任何新项目必须从 `04_生成输出/` 开始。
- 若为历史实验/测试产物，标注 `_archived_` 前缀或移入 `_归档/`，避免与主流程混淆。
- 若为正式 v2 升级，必须满足以下条件才能切换：
  1. 所有旧流程产出已迁移到新目录
  2. `规划表编写说明.md` 已更新声明新主流程路径
  3. 旧目录标注为历史参考，入口脚本指向新路径
  4. 全团队知晓并确认

### 9.4 细目表交付规则

**细目表（专题训练卷 + 课程综合卷的 docx）仍需交付。** 它是出卷的逐题约束，后续 AI 拉题/命题依赖它。

细目表生成是规划表流程的一部分（见第四章），并非独立于新旧流程的产物。每新增一个省份/考类，细目表与规划表、映射表一并产出到 `04_生成输出/生产规划/{省份} {考类}/`。

### 9.5 所有新项目统一规则

无论后续增加哪个省份/考类：

- ✅ 输出一律放在 `04_生成输出/` 对应子目录下
- ✅ 规划表 + 细目表 + 映射表放在 `04_生成输出/生产规划/{省份} {考类}/`
- ✅ 最终试卷放在 `04_生成输出/生成结果/{省份}/{考类}/`
- ❌ 不要为不同省份/考类建立独立的输出根目录
- ❌ 不要在同一项目内维护两套并行的目录拓扑
