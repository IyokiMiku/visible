# 考纲百套卷可复用代码与架构说明

> **本文档为历史参考，当前架构请以项目根目录 `README.md` 为准。**

---

## 一、主入口

| 入口 | 说明 |
|---|---|
| `create.py` | 模式选择器：1=质检修复模式，2=API拉题+质检修复模式 |
| `check.py` | 独立质检入口（双击运行，交互式选择试卷→拆题→结构化质检→输出报告） |

---

## 二、生成器模块（主链路核心）

目录：`01_工具脚本/生成器/`

| 文件 | 说明 |
|---|---|
| `runner.py` | 主流程调度中心：模式路由、交互选择、预检、卷状态、跨卷查重 |
| `planning.py` | 规划表 .xlsx 解析，生成 PaperPlan/PlanningMeta |
| `paper_loader.py` | DOCX/文本拆题为结构化 JSON |
| `paper_assembler.py` | 结构化题目 → 解析版文本组装 |
| `prompts.py` | AI 生成/修复提示词构造 |
| `regenerator.py` | 不合格题目 AI 定向修复（含答案分布本地修复） |
| `docx_generation.py` | 解析版 Word 生成 |
| `postprocess.py` | 原卷版生成 + ZIP 打包 |
| `config_io.py` | config.json 读取、API 调用封装、Token 用量统计 |
| `paths.py` | 项目路径常量 |
| `question_types.py` | 题型归一化唯一事实源（TYPE_NORMALIZE + TYPE_ORDER） |
| `image_risk.py` | 配图风险检测 |
| `_topic_pool.py` | 专题小题库管理（增删查改、去重、状态追踪） |
| `planning_validation.py` | 规划表/映射表校验 |
| `validate_planning.py` | 规划验证独立入口 |
| `quality.py` | 质量相关辅助 |
| `text_processing.py` | 文本清洗处理 |
| `exam_style.py` | 考试风格相关 |
| `references.py` | 参考资料加载 |

---

## 三、质检模块

目录：`01_工具脚本/质检/`

| 文件 | 说明 |
|---|---|
| `rules.py` | 结构化质检规则库（单题+整卷+跨卷，题型归一化委托给 question_types.py） |
| `report.py` | 质检报告 Markdown 输出 |
| `check.py` | 旧版兼容层，全部委托给 rules.py / report.py |

---

## 四、学科网 API 拉题移植版

目录：`01_工具脚本/学科网API拉题移植版/`

**已接入主流程（模式 2）的函数：**

| 文件 | 函数 | 用途 |
|---|---|---|
| `query_questions.py` | `query_questions`, `build_payload` | API 请求封装，按 courseId/typeId/kpointIds 拉题 |
| `kpoint_resolver.py` | `load_mapping_table`, `resolve_course`, `resolve_type`, `get_mapping_ai_generate_papers` | 映射表加载、课程/题型/知识点解析、AI 补题卷识别 |
| `api_pull_core.py` | `apply_content_filter` | 入库过滤（状态、知识点、难度、答案有效性） |
| `html_content_converter.py` | `convert_html_to_text` | HTML/QML 内容 → 纯文本转换 |

**独立调试/手工工具（未接入主流程）：**

| 文件 | 说明 |
|---|---|
| `api_pull_core.py`（其他函数） | `pull_for_section()`, `random_page_pull()`, `classify_question()` 等可作为手工补题工具 |
| `kpoint_type_summary.py` | 知识点题型汇总 |
| `update_type_counts.py` | 更新题型计数 |
